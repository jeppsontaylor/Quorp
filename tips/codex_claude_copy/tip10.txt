Absolutely — build **Quorp** as a **terminal-native Rust agent IDE**, not as a generic Ratatui-style widget app. The key is to treat the terminal like a high-speed, truecolor typography surface: ANSI/OSC control sequences, Unicode art, GPU-rendered terminal fonts, carefully throttled streaming, custom diff rendering, and a fast Rust agent core.

One useful reality check: Codex CLI’s current docs say it is a local terminal coding agent that can read, change, and run code, and that it is open source and built in Rust; its docs also describe a full-screen terminal UI with syntax-highlighted code blocks, diffs, approvals, and themes. ([OpenAI Developers][1]) Claude Code’s docs frame it as an agentic coding tool that reads a codebase, edits files, runs commands, and integrates with dev tools; the docs also expose the permission, hook, and subagent patterns worth copying conceptually for Quorp. ([Claude][2])

## 1. Product direction: “terminal-native IDE agent,” not “Rust TUI”

When you say the Rust TUI text feels pixelated, the issue is usually not Rust. Text sharpness comes from the terminal emulator, font, font fallback, scaling, and whether the app is drawing native glyphs or fake bitmap/block text. Quorp should render real UTF-8 text, ANSI truecolor, and carefully chosen Unicode shapes, then let terminals like WezTerm, Kitty, Ghostty, iTerm2, or Windows Terminal do the antialiasing. WezTerm, for example, advertises GPU-accelerated rendering, true color, ligatures, color emoji, and font fallback; Kitty’s graphics protocol can optionally display pixel images, but Quorp should keep the core splash text-mode so it works everywhere. ([WezTerm][3])

The design target:

**Quorp feels like a premium terminal IDE:**

* blazing startup, visible within 50–100 ms;
* black/charcoal base with molten orange, neon cyan, mint, violet, and diff red/green;
* smooth streaming with no flicker;
* tool calls as beautiful, inspectable cards;
* diffs with strong red/green background panels;
* mascot-first branding;
* unusual terminal glyphs and less-common emojis, but with ASCII fallback;
* CLI ergonomics first, optional GUI later.

## 4. Terminal rendering architecture

Do **not** start with Ratatui. Ratatui is excellent, but it tends to make apps look like terminal dashboards. Quorp should feel more like a custom interactive terminal instrument.

Use this architecture:

```text
Model stream / tool events
        ↓
Event bus: tokio::mpsc<Event>
        ↓
App state reducer
        ↓
Scene builder
        ↓
CellBuffer / DrawCommand list
        ↓
Dirty-region ANSI renderer
        ↓
Terminal
```

Important rendering rules:

1. **Use native text, not image-rendered text.** The terminal font should render the wordmark, code, and UI.
2. **Throttle streaming.** Do not repaint for every token. Coalesce to 24–60 FPS.
3. **Use dirty regions.** Track changed cells and only write the differences.
4. **Avoid `println!` during UI mode.** Use buffered `queue!` calls and flush once per frame.
5. **Use alternate screen for IDE mode.** Use normal stdout for scripted/noninteractive mode.
6. **Support resize events.** Reflow markdown, composer, and diffs.
7. **Use bracketed paste.** Pasting code should not trigger accidental submit.
8. **Use OSC 8 hyperlinks.** File paths and docs can be clickable in capable terminals.
9. **Probe terminal capabilities.** Truecolor, emoji width, Kitty/iTerm image support, hyperlink support, shell integration.

## 5. Agent and tool-calling design

Codex and Claude Code both lean heavily into controlled tool execution and approvals. Codex exposes approval controls and config-based sandbox/approval settings, while Claude Code documents a tiered permission system with allow/ask/deny rules and modes such as plan and accept-edits. ([OpenAI Developers][5])

Quorp should have these modes:

| Mode           | Behavior                                                                  |
| -------------- | ------------------------------------------------------------------------- |
| `plan`         | Read/search only. No file writes, no shell execution.                     |
| `ask`          | Ask before edits and risky commands. Default.                             |
| `accept-edits` | Auto-accept workspace edits, ask for shell/network/destructive actions.   |
| `auto-safe`    | Classify tool calls; auto-approve low-risk actions; ask/block risky ones. |
| `yolo-sandbox` | Only inside devcontainer/worktree sandbox. Never on host by default.      |

Tool categories:

| Tool            | Default permission                   |
| --------------- | ------------------------------------ |
| `read_file`     | allow                                |
| `list_files`    | allow                                |
| `search_text`   | allow                                |
| `ast_query`     | allow                                |
| `git_status`    | allow                                |
| `git_diff`      | allow                                |
| `write_file`    | ask                                  |
| `patch_file`    | ask or accept-edits                  |
| `shell`         | ask                                  |
| `run_tests`     | ask first time, remember per command |
| `network_fetch` | ask                                  |
| `mcp_call`      | ask by server/tool policy            |
| `delete_file`   | ask + scary confirmation             |
| `git_push`      | ask + scary confirmation             |
| `secret_read`   | deny by default                      |

## 6. Diff UX

The diff view should be one of Quorp’s signature features.

### Unified diff style

```text
🪩 diff lens  crates/quorp-diff/src/render.rs
╭─ hunk @@ -41,8 +41,12 @@ render_line ─────────────────────────╮
│  41 │     let width = measure(line);                                  │
│ -42 │     paint(line, Theme::default());                               │
│ -43 │     flush();                                                     │
│ +42 │     let bg = match line.kind {                                   │
│ +43 │         Add => theme.diff_add_bg,                                │
│ +44 │         Delete => theme.diff_del_bg,                             │
│ +45 │         Context => theme.diff_context_bg,                        │
│ +46 │     };                                                           │
│ +47 │     paint_with_bg(line, bg);                                     │
╰──────────────────────────── accept [a] reject [r] open [o] ─────────╯
```

### Implementation details

Use `similar` for line-level and word-level diffing. Build your own renderer over the result:

```text
DiffModel
  FileDiff
    Hunk
      Line {
        old_no,
        new_no,
        kind: Add | Delete | Context,
        spans: Vec<InlineSpan>
      }
```

Patch application should be safe:

1. Record pre-edit file hash.
2. Apply patch to a rope buffer.
3. Validate expected old lines.
4. Write atomically through temp file + rename.
5. Create checkpoint.
6. Re-run diff from disk.
7. Show final rendered hunk.

struct TerminalGuard;

impl TerminalGuard {
    fn enter() -> io::Result<Self> {
        terminal::enable_raw_mode()?;
        let mut out = stdout();
        execute!(out, EnterAlternateScreen, Hide)?;
        Ok(Self)
    }
}

impl Drop for TerminalGuard {
    fn drop(&mut self) {
        let _ = terminal::disable_raw_mode();
        let mut out = stdout();
        let _ = execute!(
            out,
            ResetColor,
            SetAttribute(Attribute::Reset),
            Show,
            LeaveAlternateScreen
        );
    }
}

fn main() -> io::Result<()> {
    let _terminal = TerminalGuard::enter()?;
    let mut out = stdout();

    run_splash(&mut out, env!("CARGO_PKG_VERSION"))
}

fn run_splash(out: &mut Stdout, version: &str) -> io::Result<()> {
    let started = Instant::now();
    let total = Duration::from_millis(2800);
    let mut frame: usize = 0;

    loop {
        let elapsed = started.elapsed();
        let progress = (elapsed.as_secs_f32() / total.as_secs_f32()).min(1.0);

        draw_splash(out, version, progress, frame)?;

        if elapsed >= total {
            break;
        }

        if event::poll(Duration::from_millis(70))? {
            if let Event::Key(key) = event::read()? {
                match key.code {
                    KeyCode::Esc | KeyCode::Enter | KeyCode::Char('q') => break,
                    _ => {}
                }
            }
        }

        frame = frame.wrapping_add(1);
    }

    Ok(())
}

fn draw_splash(out: &mut Stdout, version: &str, progress: f32, frame: usize) -> io::Result<()> {
    let (w, h) = terminal::size()?;

    queue!(
        out,
        SetBackgroundColor(ct(ABYSS)),
        Clear(ClearType::All),
        ResetColor
    )?;

    if w < 82 || h < 29 {
        draw_compact(out, version, progress, frame, w, h)?;
        out.flush()?;
        return Ok(());
    }

    let mut y = h.saturating_sub(29) / 2;

    for line in WORDMARK {
        draw_gradient_line(out, w, y, line, EMBER, HALO, true)?;
        y += 1;
    }

    y += 1;

    draw_row(
        out,
        w,
        y,
        &[
            Span::new("❯_", MOLTEN).bold(),
            Span::new("  QUORP AGENTIC IDE  ", TEXT).bold(),
            Span::new(format!(" v{} ", version), HALO).bg(PANEL).bold(),
            Span::new("  ◆  ", DIM),
            Span::new("terminal-native rust", CYAN),
        ],
    )?;

    y += 2;

    draw_gradient_line(
        out,
        w,
        y,
        "╭────────────────────────────────────────────────────────────────────╮",
        EMBER,
        HALO,
        false,
    )?;

    y += 1;

    let mascot_w = max_text_width(&MASCOT);
    let group_w = mascot_w.saturating_add(14);
    let mascot_x = w.saturating_sub(group_w) / 2 + 14;
    let prompt_x = mascot_x.saturating_sub(13);

    draw_prompt_icon(out, prompt_x, y + 4, frame)?;

    for line in MASCOT {
        draw_mascot_line(out, mascot_x, y, line, frame)?;
        y += 1;
    }

    draw_gradient_line(
        out,
        w,
        y,
        "╰────────────────────────────────────────────────────────────────────╯",
        EMBER,
        HALO,
        false,
    )?;

    y += 2;

    draw_status(out, w, y, progress, frame)?;
    y += 2;

    draw_diff_preview(out, w, y)?;
    y += 2;

    draw_row(
        out,
        w,
        y,
        &[
            Span::new("press ", DIM),
            Span::new("Enter", TEXT).bold(),
            Span::new(" to skip  ", DIM),
            Span::new("Esc", TEXT).bold(),
            Span::new(" to vanish  ", DIM),
            Span::new("🜂", EMBER),
        ],
    )?;

    out.flush()?;
    Ok(())
}

fn draw_compact(
    out: &mut Stdout,
    version: &str,
    progress: f32,
    frame: usize,
    w: u16,
    h: u16,
) -> io::Result<()> {
    let mut y = h.saturating_div(2).saturating_sub(4);

    draw_gradient_line(out, w, y, "❯_ QUORP", EMBER, HALO, true)?;
    y += 1;

    draw_row(
        out,
        w,
        y,
        &[
            Span::new("v", DIM),
            Span::new(version, HALO).bold(),
            Span::new("  terminal-native rust", CYAN),
        ],
    )?;

    y += 2;
    draw_status(out, w, y, progress, frame)?;

    y += 2;
    draw_row(
        out,
        w,
        y,
        &[Span::new("enlarge terminal for full mascot ember mode", MUTED)],
    )?;

    Ok(())
}

fn draw_prompt_icon(out: &mut Stdout, x: u16, y: u16, frame: usize) -> io::Result<()> {
    let pulse = pulse(frame, 0.20);

    for (i, line) in PROMPT_ICON.iter().enumerate() {
        let hot = lerp(EMBER, HALO, pulse);
        draw_gradient_line_at(out, x, y + i as u16, line, hot, MOLTEN, true)?;
    }

    Ok(())
}

fn draw_mascot_line(out: &mut Stdout, x: u16, y: u16, line: &str, frame: usize) -> io::Result<()> {
    let p = pulse(frame, 0.18);

    // Glow pass. This produces a slight orange rim to the right of dense shapes.
    let mut col = 0u16;
    for ch in line.chars() {
        let width = glyph_width(ch);

        if is_glow_candidate(ch) {
            queue!(
                out,
                MoveTo(x.saturating_add(col).saturating_add(1), y),
                SetForegroundColor(ct(lerp(INFERNO, EMBER, p))),
                SetAttribute(Attribute::Dim),
                Print("░"),
                ResetColor,
                SetAttribute(Attribute::Reset)
            )?;
        }

        col = col.saturating_add(width);
    }

    // Main pass.
    col = 0;
    for ch in line.chars() {
        let width = glyph_width(ch);

        if ch != ' ' {
            let fg = mascot_color(ch, p);

            queue!(out, MoveTo(x.saturating_add(col), y), SetForegroundColor(ct(fg)))?;

            if matches!(ch, '◉' | '●') {
                queue!(out, SetAttribute(Attribute::Bold))?;
            }

            queue!(
                out,
                Print(ch),
                ResetColor,
                SetAttribute(Attribute::Reset)
            )?;
        }

        col = col.saturating_add(width);
    }

    Ok(())
}

fn mascot_color(ch: char, pulse: f32) -> Rgb {
    match ch {
        '◉' | '●' => lerp(EMBER, HALO, pulse),
        'ᴗ' | '╰' | '╯' | '─' => lerp(MOLTEN, HALO, pulse * 0.35),
        '▟' | '▙' | '▜' | '▛' | '▀' | '▄' => lerp(BODY_EDGE, EMBER, pulse * 0.16),
        '█' => BODY,
        _ => BODY_EDGE,
    }
}

fn is_glow_candidate(ch: char) -> bool {
    matches!(
        ch,
        '▟' | '▙' | '▜' | '▛' | '▀' | '▄' | '◉' | '●'
    )
}

fn draw_status(out: &mut Stdout, w: u16, y: u16, progress: f32, frame: usize) -> io::Result<()> {
    let spinner = ["🜂", "🜄", "🜁", "🜃", "🪬", "🧿", "🧬", "🪩"];
    let steps = [
        "igniting ember core",
        "sealing permission gates",
        "mapping repo orbit",
        "braiding AST memory",
        "polishing diff glass",
        "readying tool spirits",
    ];

    let step_idx = ((progress * steps.len() as f32).floor() as usize).min(steps.len() - 1);
    let glyph = spinner[frame % spinner.len()];
    let bar = progress_bar(progress, 26);
    let pct = (progress * 100.0).round() as u8;

    draw_row(
        out,
        w,
        y,
        &[
            Span::new(glyph, HALO).bold(),
            Span::new("  ", TEXT),
            Span::new(steps[step_idx], TEXT).bold(),
            Span::new("   ", TEXT),
            Span::new(bar, MOLTEN).bold(),
            Span::new(format!(" {:>3}% ", pct), MUTED),
            Span::new(" tools warm ", HALO).bg(PANEL).bold(),
            Span::new(" diffs lit ", MINT).bg(PANEL).bold(),
            Span::new(" MCP gated ", VIOLET).bg(PANEL).bold(),
        ],
    )
}

fn draw_diff_preview(out: &mut Stdout, w: u16, y: u16) -> io::Result<()> {
    draw_row(
        out,
        w,
        y,
        &[
            Span::new(" − stale_guess.rs ", DIFF_DEL_FG)
                .bg(DIFF_DEL_BG)
                .bold(),
            Span::new("  ", TEXT),
            Span::new(" + focused_patch.rs ", DIFF_ADD_FG)
                .bg(DIFF_ADD_BG)
                .bold(),
            Span::new("  ", TEXT),
            Span::new("🪩 diff backgrounds online", MUTED),
        ],
    )
}

fn draw_row(out: &mut Stdout, term_w: u16, y: u16, spans: &[Span]) -> io::Result<()> {
    let width = spans
        .iter()
        .map(|s| text_width(&s.text))
        .fold(0u16, |acc, w| acc.saturating_add(w));

    let mut x = term_w.saturating_sub(width) / 2;

    for span in spans {
        queue!(out, MoveTo(x, y), SetForegroundColor(ct(span.fg)))?;

        if let Some(bg) = span.bg {
            queue!(out, SetBackgroundColor(ct(bg)))?;
        }

        if span.bold {
            queue!(out, SetAttribute(Attribute::Bold))?;
        }

        if span.dim {
            queue!(out, SetAttribute(Attribute::Dim))?;
        }

        queue!(
            out,
            Print(&span.text),
            ResetColor,
            SetAttribute(Attribute::Reset)
        )?;

        x = x.saturating_add(text_width(&span.text));
    }

    Ok(())
}

fn draw_gradient_line(
    out: &mut Stdout,
    term_w: u16,
    y: u16,
    text: &str,
    start: Rgb,
    end: Rgb,
    bold: bool,
) -> io::Result<()> {
    let width = text_width(text);
    let x = term_w.saturating_sub(width) / 2;
    draw_gradient_line_at(out, x, y, text, start, end, bold)
}

fn draw_gradient_line_at(
    out: &mut Stdout,
    x: u16,
    y: u16,
    text: &str,
    start: Rgb,
    end: Rgb,
    bold: bool,
) -> io::Result<()> {
    let total = text_width(text).max(1) as f32;
    let mut col = 0u16;

    for ch in text.chars() {
        let width = glyph_width(ch);

        if ch != ' ' {
            let t = (col as f32 / total).min(1.0);
            queue!(
                out,
                MoveTo(x.saturating_add(col), y),
                SetForegroundColor(ct(lerp(start, end, t)))
            )?;

            if bold {
                queue!(out, SetAttribute(Attribute::Bold))?;
            }

            queue!(
                out,
                Print(ch),
                ResetColor,
                SetAttribute(Attribute::Reset)
            )?;
        }

        col = col.saturating_add(width);
    }

    Ok(())
}

fn progress_bar(progress: f32, width: usize) -> String {
    let filled = ((progress.clamp(0.0, 1.0) * width as f32).round() as usize).min(width);
    format!("{}{}", "▰".repeat(filled), "▱".repeat(width - filled))
}

fn text_width(text: &str) -> u16 {
    UnicodeWidthStr::width(text).min(u16::MAX as usize) as u16
}

fn max_text_width(lines: &[&str]) -> u16 {
    lines.iter().map(|line| text_width(line)).max().unwrap_or(0)
}

fn glyph_width(ch: char) -> u16 {
    UnicodeWidthChar::width(ch).unwrap_or(1) as u16
}

fn pulse(frame: usize, speed: f32) -> f32 {
    (((frame as f32 * speed).sin() + 1.0) / 2.0).clamp(0.0, 1.0)
}

fn lerp(a: Rgb, b: Rgb, t: f32) -> Rgb {
    let t = t.clamp(0.0, 1.0);

    Rgb {
        r: mix(a.r, b.r, t),
        g: mix(a.g, b.g, t),
        b: mix(a.b, b.b, t),
    }
}

fn mix(a: u8, b: u8, t: f32) -> u8 {
    (a as f32 + (b as f32 - a as f32) * t).round() as u8
}

fn ct(c: Rgb) -> Color {
    Color::Rgb {
        r: c.r,
        g: c.g,
        b: c.b,
    }
}

## 11. The opinionated final stack

For the first production version, I would choose:

```text
Language:        Rust
Terminal core:  crossterm + custom renderer
Async:          tokio
Diffs:          similar + custom renderer
Syntax:         tree-sitter-highlight
Markdown:       comrak
Repo search:    rg child process + ignore/globset fallback
Git:            gix
PTY:            portable-pty
Storage:        rusqlite
LLM streaming:  reqwest + eventsource-stream
MCP:            JSON-RPC stdio/HTTP bridge
Plugins:        wasmtime
Config:         serde + toml + directories
Telemetry:      tracing
Testing:        insta + assert_cmd + cargo-nextest
Packaging:      cargo-dist + Homebrew + npm shim + Winget/Scoop
```

That gives you a sharp, fast, brand-heavy CLI IDE that feels closer to Claude/Codex-style terminal agents than a boxed dashboard, while still being fully Rust, fully scriptable, and extensible.
