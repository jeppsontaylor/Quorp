## Executive direction

Build QUORP as a **terminal-native agentic coding IDE**, but do **not** build the main experience as a heavy full-screen Rust TUI dashboard. Use Rust for speed, safety, streaming, patching, indexing, and tooling, while rendering through **direct ANSI/VT escape sequences, Unicode glyphs, truecolor RGB, and small cursor-controlled regions**.

A useful correction: Rust itself is not what makes text pixelated. Terminal apps do not rasterize text; the terminal emulator and font do. Codex CLI is officially described as a local terminal coding agent that can read, change, and run code, and OpenAI’s docs say it is open source and built in Rust; its docs also describe an interactive full-screen terminal UI. Claude Code is similarly documented as a full-featured CLI that works directly in the terminal. The “sharp” look comes from the terminal renderer, vector font, truecolor ANSI styling, tight redraw strategy, and restrained UI—not from avoiding Rust. ([OpenAI Developers][1])

My recommendation: **Rust core + ANSI cinematic renderer + optional editor integrations**.

# 2. Core architecture

Use a Rust workspace with a reusable agent core and multiple frontends.

```text
quorp/
  crates/
    quorp-cli/          # terminal entrypoint
    quorp-ui-ansi/      # direct ANSI renderer, no heavy TUI dependency
    quorp-agent/        # agent loop, context, model adapters
    quorp-tools/        # read_file, apply_patch, shell, git, search
    quorp-diff/         # diff engine, color renderer, patch validation
    quorp-index/        # repo map, symbols, embeddings/FTS
    quorp-mcp/          # MCP client/server support
    quorp-lsp/          # diagnostics, go-to-definition, code actions
    quorp-config/       # config, profiles, themes
    quorp-security/     # permissions, sandboxing, trust model
    quorp-telemetry/    # tracing, logs, perf counters
```

The main UI should be an **event stream renderer**, not a full dashboard.

```text
User input
   ↓
Composer / slash commands
   ↓
Agent loop
   ↓
Tool calls, model stream, diffs, approvals
   ↓
UiEvent stream
   ↓
ANSI renderer
   ↓
Terminal
```

This keeps QUORP feeling fast because each action becomes a small rendered event:

```text
🜁 thinking    scanning workspace conventions
🧿 read        crates/quorp-agent/src/loop.rs
🜃 index       912 files, 84 symbols
🪄 patch       src/main.rs  +42 −7
🧪 test        cargo test -q
🪬 approval    write outside workspace blocked
```

# 3. Recommended Rust stack

## CLI and runtime

| Need          | Recommended stack                          | Why                                          |
| ------------- | ------------------------------------------ | -------------------------------------------- |
| CLI args      | `clap`, `clap_complete`                    | Mature flags, subcommands, completions       |
| Async runtime | `tokio`                                    | Streaming model output, tools, file watchers |
| HTTP/SSE      | `reqwest`, `eventsource-stream`, `futures` | Model streaming and MCP HTTP transports      |
| Serialization | `serde`, `serde_json`, `toml`              | Config, tool schemas, transcripts            |
| Tool schemas  | `schemars`                                 | Generate JSON schema for tool calling        |
| Errors        | `anyhow`, `thiserror`, `color-eyre`        | Clean CLI diagnostics                        |
| Logging       | `tracing`, `tracing-subscriber`            | Debug without polluting user UI              |
| Cancellation  | `tokio-util::sync::CancellationToken`      | Stop model/tool execution cleanly            |
| Secrets       | `keyring`, `secrecy`, `zeroize`            | Store provider keys safely                   |

## Terminal rendering

| Need                | Recommended stack                                                    | Why                                                                        |
| ------------------- | -------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| ANSI/cursor control | `crossterm`                                                          | Cross-platform terminal control without committing to a full TUI framework |
| Styling             | raw ANSI truecolor, `anstyle`, `nu-ansi-term`, or custom RGB helpers | Precise brand colors                                                       |
| Unicode width       | `unicode-width`                                                      | Emojis and wide glyphs need correct alignment                              |
| Grapheme handling   | `unicode-segmentation`                                               | Avoid splitting composed glyphs                                            |
| Wrapping            | `textwrap`                                                           | Clean terminal-width-aware output                                          |
| Color detection     | `supports-color`, env checks for `COLORTERM=truecolor`               | Truecolor fallback                                                         |
| Markdown rendering  | `pulldown-cmark` or `comrak` + custom ANSI renderer                  | Avoid generic markdown UI blandness                                        |
| Syntax highlighting | `syntect`, `tree-sitter-highlight`                                   | Sharp code blocks                                                          |

Avoid making the main UI depend on `ratatui` unless you want an optional dashboard mode. `ratatui` is excellent, but the QUORP feel should be **streaming, cinematic, scrollback-friendly, and minimal**, not boxed-in.

## Input and command palette

| Need               | Recommended stack                             |
| ------------------ | --------------------------------------------- |
| Line editing       | `reedline` or `rustyline`                     |
| Multiline composer | `reedline` with custom keybindings            |
| Fuzzy picker       | `nucleo`, `skim`, or custom matcher           |
| Slash commands     | custom registry over `clap`-like definitions  |
| File mentions      | `ignore`, `walkdir`, `globset`, fuzzy matcher |
| History            | SQLite or JSONL in `~/.quorp/history`         |

QUORP commands should feel keyboard-first:

```text

/diff
/apply
/undo
/test
/review
/commit
/sandbox
/model

```

Codex’s official docs describe slash commands as a fast keyboard-first control surface for switching models, permissions, summarization, and similar actions; QUORP should use the same pattern, but with its own brand and interaction language. ([OpenAI Developers][2])

# 4. Model, tools, and reasoning architecture

## Agent loop

The QUORP agent loop should be explicit and inspectable:

```text
1. Receive user request.
2. Build context pack.
3. Ask model for next action.
4. If text: stream answer.
5. If tool call: validate arguments.
6. Ask for permission if needed.
7. Execute tool.
8. Render output.
9. Feed result back to model.
10. Repeat until final answer.
```

Do not expose raw hidden chain-of-thought. Instead, show **reasoning summaries and action states**:

```text
🜁 intent     locate the failing parser path
🧭 plan       inspect lexer → patch tokenizer → run focused tests
🧿 evidence   failure reproduces in parse_nested_call
🪄 action     applying minimal patch
🧪 verify     cargo test parser::nested_call
```

That gives the user confidence without dumping private model reasoning.

## Tool-call UX

Use compact cards:

```text
🧿 read
   file: crates/quorp-parser/src/token.rs
   range: 1..220
   3.8 ms

🜂 run
   cmd: cargo test parser::nested_call --quiet
   sandbox: workspace-write, network-off
   1.42 s

🪄 patch
   file: crates/quorp-parser/src/token.rs
   +14 −3
   status: awaiting review
```

## Tool registry

Each tool should declare:

```rust
pub struct ToolSpec {
    pub name: &'static str,
    pub description: &'static str,
    pub input_schema: serde_json::Value,
    pub permission: PermissionClass,
    pub risk: RiskLevel,
}
```

Suggested built-in tools:

| Tool              | Purpose                       | Permission class              |
| ----------------- | ----------------------------- | ----------------------------- |
| `read_file`       | Read a file range             | Read-only                     |
| `list_dir`        | Inspect workspace             | Read-only                     |
| `search`          | ripgrep-style search          | Read-only                     |
| `repo_map`        | Summarize project symbols     | Read-only                     |
| `shell`           | Run command                   | Approval depending on command |
| `apply_patch`     | Apply validated unified patch | Write approval                |
| `create_file`     | Add file                      | Write approval                |
| `delete_file`     | Delete file                   | High-risk approval            |
| `git_diff`        | Show diff                     | Read-only                     |
| `git_status`      | Show status                   | Read-only                     |
| `git_checkpoint`  | Create safety checkpoint      | Write approval                |
| `lsp_diagnostics` | Pull editor diagnostics       | Read-only                     |
| `mcp_call`        | Call external MCP tool        | Depends on server/tool        |

Anthropic documents Claude tool use as structured tool calls that your application executes and then returns as tool results; this is exactly the pattern QUORP should implement internally. ([Claude API Docs][3])

## MCP support

Support MCP early. It is now table-stakes for agentic coding tools. Codex docs state that MCP connects models to third-party tools and context and that Codex supports MCP servers in both CLI and IDE extension contexts; the MCP spec defines resources, prompts, and tools as server features. ([OpenAI Developers][4])

Implement:

```text
quorp mcp add github --transport http ...
quorp mcp add figma --transport http ...
quorp mcp add local-docs --transport stdio -- npx some-server
quorp mcp list
quorp mcp test github
```

MCP tool cards should look distinct:

```text
🪬 mcp:github.search_issues
   query: "crash on startup label:bug"
   auth: bearer-token
   result: 12 issues
```

## Hooks

Add lifecycle hooks modeled around agent events:

```text
UserPromptSubmit
PreToolUse
PermissionRequest
PostToolUse
PatchPreview
PatchApplied
TestsFinished
Stop
```

Codex and Claude Code both document hooks as a way to run scripts at lifecycle points; Codex’s docs specifically describe hooks around events such as pre-tool use, permission requests, post-tool use, prompt submit, and stop. ([OpenAI Developers][5])

Example:

```toml
[hooks.pre_tool_use.shell]
command = "quorp-secret-scan --stdin"

[hooks.patch_preview]
command = "cargo fmt --check"

[hooks.stop]
command = "quorp-memory-summarize"
```

## Diff rendering style

Use compact unified diffs by default:

```text
╭─ 🪄 patch  crates/quorp-parser/src/token.rs
│
│ @@ -42,7 +42,8 @@ fn parse_token(...)
│  42   match ch {
│- 43       '"' => parse_string(input),
│+ 43       '"' if escaped => parse_escaped_string(input),
│+ 44       '"' => parse_string(input),
│  45       _ => parse_atom(input),
│
╰─ +2 −1  awaiting approval
```

For wide terminals, add optional side-by-side mode:

```text
quorp /diff --side-by-side
```

Use `similar` or `dissimilar` for inline changes, and `gix` or `git2` for repository state. For applying edits, use a strict patch validator:

```text
- Check file is inside workspace.
- Check old lines match exactly.
- Reject overlapping patches.
- Reject binary edits unless using binary tool.
- Create git checkpoint before applying.
- Render before/after diff.
- Apply.
- Re-read file hash.
- Run requested tests.
```

# 8. Sharpness strategy

To make QUORP look crisp:

1. **Use direct ANSI truecolor**, not raster images.
2. **Use vector-friendly Unicode**: box drawing, block glyphs, braille, half-blocks.
3. **Avoid sixel/inline PNG as the default**; that can look pixelated or fail across terminals.
4. **Recommend modern terminals**: Ghostty, WezTerm, Kitty, iTerm2, Alacritty, Windows Terminal.
5. **Recommend vector coding fonts**: JetBrains Mono, Monaspace, Cascadia Code, Berkeley Mono, Fira Code.
6. **Implement `quorp doctor`** to test:

   * truecolor support
   * emoji width
   * Nerd Font glyphs
   * background contrast
   * terminal dimensions
   * shell integration
   * sandbox availability

The app cannot force the terminal to render text beautifully. It can, however, detect poor conditions and recommend fixes.

The load screen should:

* Clear the terminal.
* Hide cursor during boot.
* Center a gradient QUORP logo.
* Render a Unicode mascot inspired by the uploaded image.
* Pulse the orange eyes and embers.

    thread,

use unicode_width::UnicodeWidthStr;

const RESET: &str = "\x1b[0m";
const BOLD: &str = "\x1b[1m";
const DIM: &str = "\x1b[2m";

#[derive(Clone, Copy)]
struct Rgb(u8, u8, u8);

impl Rgb {
    fn fg(self) -> String {
        format!("\x1b[38;2;{};{};{}m", self.0, self.1, self.2)
    }

    fn lerp(a: Rgb, b: Rgb, t: f32) -> Rgb {
        let t = t.clamp(0.0, 1.0);

        let r = a.0 as f32 + (b.0 as f32 - a.0 as f32) * t;
        let g = a.1 as f32 + (b.1 as f32 - a.1 as f32) * t;
        let bl = a.2 as f32 + (b.2 as f32 - a.2 as f32) * t;

        Rgb(r.round() as u8, g.round() as u8, bl.round() as u8)
    }
}

struct CursorGuard;

impl Drop for CursorGuard {
    fn drop(&mut self) {
        let _ = execute!(io::stdout(), Show);
    }
}

fn main() -> io::Result<()> {
    let _cursor_guard = CursorGuard;
    let mut out = io::stdout();

    execute!(out, Hide)?;

    let total_frames = 34;

    for frame in 0..=total_frames {
        let (cols, rows) = terminal::size().unwrap_or((100, 34));

        execute!(out, MoveTo(0, 0), Clear(ClearType::All))?;
        render_screen(&mut out, cols, rows, frame, total_frames)?;
        out.flush()?;

        thread::sleep(Duration::from_millis(38));
    }

    execute!(out, Show)?;
    Ok(())
}

fn render_screen(
    out: &mut impl Write,
    cols: u16,
    rows: u16,
    frame: usize,
    total_frames: usize,
) -> io::Result<()> {
    let full_layout = cols >= 88;
    let content_height = if full_layout { 26 } else { 10 };
    let top_pad = rows.saturating_sub(content_height) / 2;

    for _ in 0..top_pad {
        writeln!(out)?;
    }

    if full_layout {
        for line in LOGO {
            gradient_centered(out, cols, line, color::ORANGE, color::GOLD, true)?;
        }

        writeln!(out)?;

        for line in MASCOT {
            mascot_centered(out, cols, line, frame)?;
        }
    } else {
        gradient_centered(
            out,
            cols,
            "❯_  QUORP  ◉ • ◉",
            color::ORANGE,
            color::GOLD,
            true,
        )?;

        centered(
            out,
            cols,
            "molten coding familiar",
            color::MUTED,
            false,
            false,
        )?;
    }

    writeln!(out)?;

    let version = format!(
        "QUORP v{}  •  molten terminal IDE  •  {}-{}",
        env!("CARGO_PKG_VERSION"),
        env::consts::OS,
        env::consts::ARCH
    );

    centered(out, cols, &version, color::TEXT, true, false)?;

    centered(
        out,
        cols,
        "sharp ANSI • glowing diffs • fast tools • guarded edits",
        color::MUTED,
        false,
        true,
    )?;

    writeln!(out)?;

    render_progress(out, cols, frame, total_frames)?;

    Ok(())
}

fn centered(
    out: &mut impl Write,
    cols: u16,
    text: &str,
    fg: Rgb,
    bold: bool,
    dim: bool,
) -> io::Result<()> {
    let width = UnicodeWidthStr::width(text);
    let pad = (cols as usize).saturating_sub(width) / 2;

    write!(out, "{}", " ".repeat(pad))?;

    if bold {
        write!(out, "{BOLD}")?;
    }

    if dim {
        write!(out, "{DIM}")?;
    }

    writeln!(out, "{}{}{}", fg.fg(), text, RESET)?;
    Ok(())
}

fn gradient_centered(
    out: &mut impl Write,
    cols: u16,
    text: &str,
    from: Rgb,
    to: Rgb,
    bold: bool,
) -> io::Result<()> {
    let width = UnicodeWidthStr::width(text);
    let pad = (cols as usize).saturating_sub(width) / 2;

    write!(out, "{}", " ".repeat(pad))?;

    if bold {
        write!(out, "{BOLD}")?;
    }

    let chars: Vec<char> = text.chars().collect();
    let denom = chars.len().saturating_sub(1).max(1) as f32;

    for (idx, ch) in chars.iter().enumerate() {
        if *ch == ' ' {
            write!(out, " ")?;
            continue;
        }

        let t = idx as f32 / denom;
        let fg = Rgb::lerp(from, to, t);
        write!(out, "{}{}", fg.fg(), ch)?;
    }

    writeln!(out, "{RESET}")?;
    Ok(())
}

fn mascot_centered(
    out: &mut impl Write,
    cols: u16,
    line: &str,
    frame: usize,
) -> io::Result<()> {
    let width = UnicodeWidthStr::width(line);
    let pad = (cols as usize).saturating_sub(width) / 2;

    write!(out, "{}", " ".repeat(pad))?;

    for ch in line.chars() {
        if ch == ' ' {
            write!(out, " ")?;
            continue;
        }

        let fg = mascot_color(ch, frame);
        write!(out, "{}{}{}", fg.fg(), ch, RESET)?;
    }

    writeln!(out)?;
    Ok(())
}

fn mascot_color(ch: char, frame: usize) -> Rgb {
    let pulse = ember_pulse(frame);

    match ch {
        '◉' | '●' => pulse,
        '❯' | '_' => Rgb::lerp(color::ORANGE, color::GOLD, 0.35),
        '•' => color::EMBER,
        '✧' | '✦' | '✹' | '⠁' | '⠂' => Rgb::lerp(color::GOLD, color::EMBER, 0.25),
        '╭' | '╮' | '╰' | '╯' | '╲' | '╱' | '─' | '│' => color::BODY,
        _ => color::BODY_DARK,
    }
}

fn ember_pulse(frame: usize) -> Rgb {
    let cycle = (frame % 18) as f32 / 17.0;
    let wave = if cycle < 0.5 {
        cycle * 2.0
    } else {
        (1.0 - cycle) * 2.0
    };

    Rgb::lerp(color::ORANGE, color::GOLD, wave)
}

fn render_progress(
    out: &mut impl Write,
    cols: u16,
    frame: usize,
    total_frames: usize,
) -> io::Result<()> {
    let pct = ((frame as f32 / total_frames as f32) * 100.0).round() as usize;

    let phase = match pct {
        0..=20 => "igniting prompt sigil",
        21..=40 => "polishing molten glyphs",
        41..=60 => "priming tool lattice",
        61..=80 => "arming sandbox wards",
        _ => "awakening diff engine",
    };

    let rune = RUNES[frame % RUNES.len()];
    let bar_width = (cols as usize).saturating_sub(42).clamp(18, 48);
    let filled = ((pct as f32 / 100.0) * bar_width as f32).round() as usize;

    let label = format!("{}  {}  {:>3}%", rune, phase, pct);
    let visible_width = UnicodeWidthStr::width(label.as_str()) + bar_width + 4;
    let pad = (cols as usize).saturating_sub(visible_width) / 2;

    write!(out, "{}", " ".repeat(pad))?;

    write!(out, "{}{}{}  ", color::GOLD.fg(), rune, RESET)?;
    write!(out, "{}[", color::GHOST.fg())?;

    for idx in 0..bar_width {
        if idx < filled {
            let t = idx as f32 / bar_width.saturating_sub(1).max(1) as f32;
            let fg = Rgb::lerp(color::LAVA, color::GOLD, t);
            write!(out, "{}━", fg.fg())?;
        } else {
            write!(out, "{}·", color::GHOST.fg())?;
        }
    }

    write!(
        out,
        "{}]{}  {}{}{} {:>3}%{}",
        color::GHOST.fg(),
        RESET,
        DIM,
        color::MUTED.fg(),
        phase,
        pct,
        RESET
    )?;

    writeln!(out)?;
    Ok(())
}
