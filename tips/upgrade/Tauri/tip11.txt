## Recommendation: build Quorp Desktop as a new Tauri 2 app at `apps/desktop`

The strongest path is:

```text
apps/desktop/                 # Tauri + React/TypeScript desktop product
  package.json
  index.html
  src/
    app/
    components/
    features/
    lib/
    styles/
  src-tauri/
    Cargo.toml
    tauri.conf.json
    capabilities/
    src/

crates/quorp_desktop_core/     # Rust desktop service layer, no Tauri dependency
crates/quorp_desktop_ipc/      # serde IPC DTOs shared by Tauri commands/events
```

Do **not** put the DMG app inside `crates/quorp`. That crate is currently the thin CLI binary wrapper calling `quorp_cli::dispatch()`; keeping the CLI stable is exactly what we want. The repository already has a serious Rust workspace with `quorp_cli`, `quorp_session`, `quorp_agent_core`, `quorp_permissions`, `quorp_sandbox`, `quorp_render`, `quorp_slash`, `quorp_verify`, and related crates, so the desktop app should be another product surface over the same core rather than a fork of the CLI. The root workspace already uses explicit members and shared crate dependencies, so adding a Tauri crate plus two small desktop-specific crates fits cleanly. 

One note: I could not clone GitHub directly from the local container because DNS resolution failed, so I inspected the repo through the GitHub connector and web access instead. The repo evidence is still concrete: Quorp is currently a TUI-first Rust terminal binary, with CLI extracted into `quorp_cli`, session/runtime into `quorp_session`, typed runtime actions, Patch VM receipts, runtime event fanout, verification proof packets, and benchmark gates already present.

# 1. Product architecture

## The desktop stack

Use:

```text
Tauri 2
React
TypeScript
Vite
TanStack Virtual
Monaco Editor, lazy-loaded
xterm.js, lazy-loaded only for terminal panes
Zustand or a tiny custom external-store layer
CSS variables + CSS modules, no heavy component framework
Rust desktop core
Existing Quorp crates
```

Tauri is the right choice because it lets Quorp keep Rust as the privileged backend while using a modern web GUI for the high-density interface. Tauri supports frontend frameworks while app logic lives in Rust, and its native web renderer approach keeps apps smaller than Electron-style desktop apps. ([Tauri][1])

Do **not** use WebSockets internally. Use Tauri commands for request/response and Tauri channels for streaming runtime events. Tauri’s IPC command model is type-oriented, async commands keep the UI from freezing, and Tauri channels are specifically documented for streaming data. WebSockets should only appear later if Quorp adds a remote browser client or headless daemon mode. ([Tauri][2])

Events are fine for global notifications, but for the agent stream use channels because Quorp needs high-volume, ordered, typed-ish streams: model requests, tool calls, diffs, validation, permission prompts, proof events, and cancellation updates. Tauri’s docs distinguish channels from app events; events are global and not type-safe, while channels are the better streaming primitive. ([Tauri][2])

## Why React/TypeScript, despite Quorp being Rust-first

The core remains Rust. The UI should not.

A Codex-like desktop app needs fast virtualized scrollback, split panes, command palette, diff rendering, settings, tabs, lazy code editors, and rich keyboard interactions. React + TypeScript + Vite gets that shipped quickly and cleanly. A pure Rust GUI like Slint or egui would be attractive for purity, but it would slow down the “brilliant desktop IDE” path and make sophisticated diff/chat/sidebar/editor interactions harder to iterate.

The right split is:

```text
Rust:
  agent runtime
  permission model
  sandbox policy
  provider calls
  session persistence
  proof receipts
  subprocess/PTY handling
  file access
  secrets/keychain
  trust policy

TypeScript:
  layout
  command palette
  virtualized event feed
  diff viewer
  settings UI
  keyboard workflows
  status dashboards
  visual state
```

Monaco should be lazy-loaded because it is powerful but heavy. Use it for opened files and full diff review. For the main agent stream, render lightweight custom diff cards with virtualized rows instead of mounting Monaco for every patch. xterm.js should also be lazy-loaded and used only for true terminal sessions; it is a proven browser terminal component used by tools such as VS Code and has optional GPU/WebGL support. ([GitHub][3])

```rust
// crates/quorp_desktop_ipc/src/quorp_desktop_ipc.rs

use std::path::PathBuf;

use quorp_agent_core::{RuntimeEvent, StopReason};
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct WorkspaceId(pub String);

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RunId(pub String);

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum WorkspaceTrust {
    Untrusted,
    ReadOnlyTrusted,
    SandboxTrusted,
    FullyTrusted,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct WorkspaceSummary {
    pub id: WorkspaceId,
    pub display_name: String,
    pub root: PathBuf,
    pub trust: WorkspaceTrust,
    pub git_branch: Option<String>,
    pub dirty_files: usize,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum DesktopPermissionMode {
    ReadOnly,
    Ask,
    AcceptEdits,
    AutoSafe,
    YoloSandbox,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum DesktopSandboxMode {
    Host,
    TmpCopy,
    GitWorktreePreferred,
    ContainerNoNetwork,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AgentStartRequest {
    pub workspace_id: WorkspaceId,
    pub prompt: String,
    pub model_id: Option<String>,
    pub provider: Option<String>,
    pub permission_mode: DesktopPermissionMode,
    pub sandbox_mode: DesktopSandboxMode,
    pub max_steps: Option<usize>,
    pub max_seconds: Option<u64>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AgentStartResponse {
    pub run_id: RunId,
    pub result_dir: PathBuf,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct StatusSnapshot {
    pub active_run_id: Option<RunId>,
    pub permission_mode: DesktopPermissionMode,
    pub sandbox_mode: DesktopSandboxMode,
    pub context_used_tokens_estimate: u64,
    pub context_budget_tokens: u64,
    pub compaction_recommended: bool,
    pub total_billed_tokens: u64,
    pub current_tool: Option<String>,
    pub last_validation: Option<String>,
    pub dirty_files: usize,
    pub pending_permission_count: usize,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "event", rename_all = "snake_case")]
pub enum DesktopEvent {
    WorkspaceAdded {
        workspace: WorkspaceSummary,
    },
    WorkspaceUpdated {
        workspace: WorkspaceSummary,
    },
    RunStarted {
        run_id: RunId,
        workspace_id: WorkspaceId,
    },
    Runtime {
        run_id: RunId,
        event: RuntimeEvent,
    },
    Status {
        snapshot: StatusSnapshot,
    },
    PermissionRequested {
        request_id: String,
        run_id: RunId,
        tool_name: String,
        summary: String,
        risk_tokens: Vec<String>,
    },
    RunFinished {
        run_id: RunId,
        reason: StopReason,
    },
    Error {
        title: String,
        detail: String,
    },
}
```

This lets the frontend ingest one stream format instead of reverse-engineering CLI text.

The desktop core should expose an app service:

```rust
// crates/quorp_desktop_core/src/quorp_desktop_core.rs

use std::collections::BTreeMap;
use std::path::{Path, PathBuf};
use std::sync::{Arc, Mutex};

use anyhow::Context as _;
use quorp_desktop_ipc::{
    AgentStartRequest, AgentStartResponse, DesktopEvent, WorkspaceId, WorkspaceSummary,
    WorkspaceTrust,
};

pub trait DesktopEventEmitter: Send + Sync + 'static {
    fn emit(&self, event: DesktopEvent);
}

#[derive(Debug, Default)]
struct DesktopState {
    workspaces: BTreeMap<String, WorkspaceSummary>,
}

#[derive(Clone)]
pub struct DesktopAppService {
    state: Arc<Mutex<DesktopState>>,
    emitter: Arc<dyn DesktopEventEmitter>,
}

impl DesktopAppService {
    pub fn new(emitter: Arc<dyn DesktopEventEmitter>) -> Self {
        Self {
            state: Arc::new(Mutex::new(DesktopState::default())),
            emitter,
        }
    }

    pub fn add_workspace(&self, root: PathBuf) -> anyhow::Result<WorkspaceSummary> {
        let root = std::fs::canonicalize(&root)
            .with_context(|| format!("failed to canonicalize workspace {}", root.display()))?;

        let display_name = root
            .file_name()
            .and_then(|name| name.to_str())
            .unwrap_or("workspace")
            .to_string();

        let id = WorkspaceId(workspace_id_for_path(&root));

        let summary = WorkspaceSummary {
            id: id.clone(),
            display_name,
            root,
            trust: WorkspaceTrust::Untrusted,
            git_branch: None,
            dirty_files: 0,
        };

        {
            let mut state = self.state.lock().expect("desktop state lock");
            state.workspaces.insert(id.0.clone(), summary.clone());
        }

        self.emitter.emit(DesktopEvent::WorkspaceAdded {
            workspace: summary.clone(),
        });

        Ok(summary)
    }

    pub async fn start_agent_run(
        &self,
        request: AgentStartRequest,
    ) -> anyhow::Result<AgentStartResponse> {
        // First production version should use native runtime wiring.
        // Fast MVP can route through a ProcessAgentBackend behind the same trait.
        start_agent_run_native(self.clone(), request).await
    }
}

fn workspace_id_for_path(path: &Path) -> String {
    use sha2::{Digest, Sha256};

    let mut hasher = Sha256::new();
    hasher.update(path.to_string_lossy().as_bytes());
    format!("{:x}", hasher.finalize())
}

async fn start_agent_run_native(
    _service: DesktopAppService,
    request: AgentStartRequest,
) -> anyhow::Result<AgentStartResponse> {
    let run_id = quorp_desktop_ipc::RunId(format!(
        "desktop-{}",
        chrono::Utc::now().timestamp_millis()
    ));

    let result_dir = dirs::data_dir()
        .unwrap_or_else(|| PathBuf::from("."))
        .join("quorp")
        .join("desktop-runs")
        .join(&run_id.0);

    std::fs::create_dir_all(&result_dir)
        .with_context(|| format!("failed to create {}", result_dir.display()))?;

    // TODO: construct AgentRunRequest and RuntimeEventFanout.
    // Keep this function small; split native backend into src/agent/native.rs.

    Ok(AgentStartResponse { run_id, result_dir })
}
```

The key design choice: introduce an `AgentBackend` trait with two implementations.

```rust
pub trait AgentBackend: Send + Sync {
    fn start(
        &self,
        request: AgentStartRequest,
        emitter: Arc<dyn DesktopEventEmitter>,
    ) -> anyhow::Result<AgentStartResponse>;
}

pub struct ProcessAgentBackend;
pub struct NativeAgentBackend;
```

For **fast usability**, `ProcessAgentBackend` can spawn the existing `quorp run`/`quorp exec` path and tail `events.jsonl`. That works because Quorp’s headless runs already emit `request.json`, `metadata.json`, `summary.json`, `events.jsonl`, `proof-receipt.json`, and logs. 

For the **real product**, `NativeAgentBackend` should call directly into the runtime/session stack and forward `RuntimeEvent` through a `RuntimeEventSink`. The runtime already has typed `RuntimeEvent`, `AgentRunRequest`, `AgentRunOutcome`, `CompletionClient`, `ToolExecutor`, `RuntimeEventSink`, and `RuntimeEventFanout`; this is exactly what a desktop bridge wants.

## `apps/desktop/src-tauri/src/lib.rs`

```rust
use std::path::PathBuf;
use std::sync::Arc;

use quorp_desktop_core::{DesktopAppService, DesktopEventEmitter};
use quorp_desktop_ipc::{
    AgentStartRequest, AgentStartResponse, DesktopEvent, StatusSnapshot, WorkspaceSummary,
};
use tauri::{AppHandle, Manager};
use tauri::ipc::Channel;

#[derive(Clone)]
struct AppEmitter {
    app: AppHandle,
}

impl DesktopEventEmitter for AppEmitter {
    fn emit(&self, event: DesktopEvent) {
        if let Err(error) = self.app.emit("desktop-event", event) {
            log::warn!("failed to emit desktop event: {error}");
        }
    }
}

#[derive(Clone)]
struct AppState {
    service: DesktopAppService,
}

#[tauri::command]
async fn workspace_add(
    state: tauri::State<'_, AppState>,
    path: PathBuf,
) -> Result<WorkspaceSummary, String> {
    state
        .service
        .add_workspace(path)
        .map_err(|error| format!("{error:#}"))
}

#[tauri::command]
async fn agent_start(
    state: tauri::State<'_, AppState>,
    request: AgentStartRequest,
    events: Channel<DesktopEvent>,
) -> Result<AgentStartResponse, String> {
    // For high-volume run events, prefer this per-run Channel over global app events.
    let response = state
        .service
        .start_agent_run(request)
        .await
        .map_err(|error| format!("{error:#}"))?;

    if let Err(error) = events.send(DesktopEvent::RunStarted {
        run_id: response.run_id.clone(),
        workspace_id: quorp_desktop_ipc::WorkspaceId("pending".to_string()),
    }) {
        log::warn!("failed to send run-start event over channel: {error}");
    }

    Ok(response)
}

#[tauri::command]
async fn status_snapshot(
    _state: tauri::State<'_, AppState>,
) -> Result<StatusSnapshot, String> {
    // TODO: wire to DesktopAppService.
    Err("status_snapshot not wired yet".to_string())
}

pub fn run() {
    tauri::Builder::default()
        .plugin(tauri_plugin_dialog::init())
        .plugin(tauri_plugin_opener::init())
        .plugin(tauri_plugin_window_state::Builder::default().build())
        .setup(|app| {
            let emitter = Arc::new(AppEmitter {
                app: app.handle().clone(),
            });

            app.manage(AppState {
                service: DesktopAppService::new(emitter),
            });

            Ok(())
        })
        .invoke_handler(tauri::generate_handler![
            workspace_add,
            agent_start,
            status_snapshot,
        ])
        .run(tauri::generate_context!())
        .expect("failed to run Quorp Desktop");
}
```

Important repo rule: avoid `let _ =` on fallible operations. The handoff explicitly calls that out, along with the 2,000 LOC cap and no `mod.rs` paths. Keep every desktop file under that cap from day one.

## Local testing without delaying on signing

For fast testing on your Mac:

```bash
cd apps/desktop
pnpm install
pnpm desktop:dev
```

Then:

```bash
pnpm desktop:debug-app
```

For a local DMG:

```bash
pnpm desktop:dmg
open apps/desktop/src-tauri/target/release/bundle/dmg
```

Unsigned builds are fine for local iteration, but do not distribute them as the public release. When downloaded from the internet, unsigned or improperly signed macOS apps hit Gatekeeper friction. Tauri’s signing docs explicitly call out Developer ID signing to avoid the “broken” warning, and Apple’s notarization flow is the standard outside-App-Store trust path. ([Tauri][6])

For local-only ad hoc signing:

```bash
codesign --force --deep --sign - \
  apps/desktop/src-tauri/target/release/bundle/macos/Quorp.app
```

For release signing later:

```bash
security find-identity -v -p codesigning

export APPLE_SIGNING_IDENTITY="Developer ID Application: Your Name (TEAMID)"
export APPLE_ID="you@example.com"
export APPLE_TEAM_ID="TEAMID"
export APPLE_PASSWORD="app-specific-password"

cd apps/desktop
pnpm tauri build -- --bundles dmg

xcrun notarytool submit \
  src-tauri/target/release/bundle/dmg/Quorp_*.dmg \
  --apple-id "$APPLE_ID" \
  --team-id "$APPLE_TEAM_ID" \
  --password "$APPLE_PASSWORD" \
  --wait

xcrun stapler staple src-tauri/target/release/bundle/dmg/Quorp_*.dmg
spctl -a -vvv -t install src-tauri/target/release/bundle/dmg/Quorp_*.dmg
```

Also add macOS PATH repair immediately. GUI apps launched from Finder often do not inherit shell dotfile PATHs, and the Tauri docs call this out directly. Quorp must reliably find `git`, `cargo`, `rustup`, `node`, `python`, and other project tools from the desktop app. ([Tauri][5])

```ts
// apps/desktop/src/lib/ipc.ts

import { invoke, Channel } from "@tauri-apps/api/core";

export type WorkspaceId = { 0: string };
export type RunId = { 0: string };

export type DesktopPermissionMode =
  | "read_only"
  | "ask"
  | "accept_edits"
  | "auto_safe"
  | "yolo_sandbox";

export type DesktopSandboxMode =
  | "host"
  | "tmp_copy"
  | "git_worktree_preferred"
  | "container_no_network";

export interface AgentStartRequest {
  workspace_id: WorkspaceId;
  prompt: string;
  model_id?: string | null;
  provider?: string | null;
  permission_mode: DesktopPermissionMode;
  sandbox_mode: DesktopSandboxMode;
  max_steps?: number | null;
  max_seconds?: number | null;
}

export interface AgentStartResponse {
  run_id: RunId;
  result_dir: string;
}

export type DesktopEvent =
  | { event: "workspace_added"; workspace: unknown }
  | { event: "workspace_updated"; workspace: unknown }
  | { event: "run_started"; run_id: RunId; workspace_id: WorkspaceId }
  | { event: "runtime"; run_id: RunId; event: RuntimeEvent }
  | { event: "status"; snapshot: unknown }
  | { event: "permission_requested"; request_id: string; run_id: RunId; tool_name: string; summary: string; risk_tokens: string[] }
  | { event: "run_finished"; run_id: RunId; reason: string }
  | { event: "error"; title: string; detail: string };

export interface RuntimeEvent {
  event: string;
  [key: string]: unknown;
}

export async function addWorkspace(path: string) {
  return invoke("workspace_add", { path });
}

export async function startAgentRun(
  request: AgentStartRequest,
  onEvent: (event: DesktopEvent) => void,
): Promise<AgentStartResponse> {
  const channel = new Channel<DesktopEvent>();
  channel.onmessage = onEvent;

  return invoke<AgentStartResponse>("agent_start", {
    request,
    events: channel,
  });
}
```

## Store

```ts
// apps/desktop/src/features/agent/agentStore.ts

import { create } from "zustand";
import type { DesktopEvent, RunId } from "../../lib/ipc";

export interface AgentStreamItem {
  id: string;
  runId: string;
  kind: string;
  createdAt: number;
  payload: unknown;
}

interface AgentState {
  activeRunId: string | null;
  items: AgentStreamItem[];
  ingest: (event: DesktopEvent) => void;
  clear: () => void;
}

const MAX_RETAINED_ITEMS = 5_000;

export const useAgentStore = create<AgentState>((set) => ({
  activeRunId: null,
  items: [],
  ingest: (event) => {
    set((state) => {
      const next = eventToItem(event);
      if (!next) return state;

      const items =
        state.items.length >= MAX_RETAINED_ITEMS
          ? state.items.slice(state.items.length - MAX_RETAINED_ITEMS + 1)
          : state.items.slice();

      items.push(next);

      return {
        activeRunId: next.runId,
        items,
      };
    });
  },
  clear: () => set({ activeRunId: null, items: [] }),
}));

function eventToItem(event: DesktopEvent): AgentStreamItem | null {
  if (event.event !== "runtime") return null;

  const runId = unwrapRunId(event.run_id);

  return {
    id: `${runId}:${Date.now()}:${Math.random().toString(16).slice(2)}`,
    runId,
    kind: String(event.event.event ?? "runtime"),
    createdAt: Date.now(),
    payload: event.event,
  };
}

function unwrapRunId(runId: RunId): string {
  return runId[0];
}
```

# 8. Wiring runtime events into desktop

Quorp’s runtime already has the right primitives:

```text
AgentRunRequest
AgentRunOutcome
RuntimeEvent
RuntimeEventSink
RuntimeEventFanout
CompletionClient
ToolExecutor
AgentCheckpoint
AgentTaskStateSnapshot
```

That means the desktop app should not parse ANSI renderer output. The existing `quorp_render` crate is valuable design evidence and may provide palette/session concepts, but the desktop should subscribe to semantic runtime events and render native web UI components. `quorp_render` is explicitly unit-testable rendering logic and does not directly touch stdout, which is good, but it is still optimized for terminal output. 

The desktop bridge should look like:

```text
React composer
  ↓ invoke("agent_start")
Tauri command
  ↓
quorp_desktop_core::DesktopAppService
  ↓
NativeAgentBackend
  ↓
quorp_agent_core runtime
  ↓ RuntimeEventSink
Tauri Channel<DesktopEvent>
  ↓
React virtualized event stream
```

Implement:

```rust
use std::sync::Arc;

use quorp_agent_core::{RuntimeEvent, RuntimeEventSink};
use quorp_desktop_ipc::{DesktopEvent, RunId};

pub struct ChannelRuntimeSink {
    run_id: RunId,
    sender: tauri::ipc::Channel<DesktopEvent>,
}

impl RuntimeEventSink for ChannelRuntimeSink {
    fn emit(&self, event: RuntimeEvent) {
        let desktop_event = DesktopEvent::Runtime {
            run_id: self.run_id.clone(),
            event,
        };

        if let Err(error) = self.sender.send(desktop_event) {
            log::warn!("failed to forward runtime event to desktop channel: {error}");
        }
    }
}
```

For high volume, batch events every 16–50 ms:

```rust
pub struct BatchedRuntimeSink {
    run_id: RunId,
    sender: tauri::ipc::Channel<Vec<DesktopEvent>>,
    buffer: parking_lot::Mutex<Vec<DesktopEvent>>,
}
```

Start with unbatched for MVP; add batching once the UI is streaming.

# 12. Important implementation risks

## 1. Do not expose benchmark-specific repair behavior as a desktop default

While inspecting runtime modules through GitHub, I saw benchmark-repair/playbook-oriented runtime code in the runtime path. That may be useful for benchmark harnesses, but desktop full-auto behavior must be general. Add a visible runtime flag:

```rust
pub enum RuntimeProfile {
    GeneralDesktop,
    BenchmarkHarness,
}
```

Desktop should use `GeneralDesktop`.

## 2. Do not trust repo-authored desktop settings by default

A malicious repo can contain config, instructions, scripts, and tool outputs. Desktop trust state should live in the app config directory, not inside the repo. Project config can be read only after the user grants trust.

## 3. Do not run full-auto on host

The desktop UI should make unsafe combinations impossible:

```text
YoloSandbox + Host = disabled
AutoSafe + Host = allowed only after trust prompt
ReadOnly + Host = okay for read-only
Ask + Host = okay with prompts
```

## 4. Do not stream unbounded DOM

The agent stream must be virtualized. Keep raw events in Rust/run artifacts and use a bounded frontend store.

## 5. Do not make the frontend parse terminal ANSI

Use semantic events. ANSI rendering is for CLI/TUI. Desktop needs richer typed cards.
