## Recommendation: add a real Tauri desktop app at `apps/desktop`, plus a small shared desktop-core crate

The best location is:

```text
/apps/desktop
  Tauri 2 desktop app
  React + TypeScript + Vite frontend
  src-tauri Rust shell
  macOS bundle/DMG/signing config

/crates/quorp_desktop_core
  desktop session controller
  typed desktop events
  workspace registry
  settings/status/diff/proof DTOs
  bridge from Quorp runtime events to desktop UI events
```

Do **not** put the desktop app inside `crates/quorp_cli`. The current CLI binary is intentionally thin: `crates/quorp/src/main.rs` only calls `quorp_cli::dispatch()`, while `quorp_cli` owns clap/crossterm/terminal command dispatch. That is perfect for CLI, but it is the wrong boundary for a desktop app. The desktop should depend on `quorp_session`, `quorp_agent_core`, `quorp_config`, `quorp_context`, `quorp_sandbox`, `quorp_verify`, `quorp_slash`, and related crates directly. The repo already has a reusable session layer: `quorp_session` is described as the “session engine and chat-service runtime,” and it exposes `agent_runner`, `executor`, `prompt_compaction`, `run_support`, and `tui` services.     

The product stack I would ship first is:

```text
Tauri 2
React + TypeScript + Vite
Rust backend/session controller
Tauri commands for request/response IPC
Tauri events for streaming runtime events
No local web server
No WebSocket for local desktop IPC
Virtualized timeline/log rendering
CodeMirror 6 for diff/code panes
Lazy-loaded Monaco only if you later need full IDE editing
```

Tauri’s IPC model already gives you JSON-serializable commands and fire-and-forget events between Rust core and the WebView, so WebSockets are unnecessary for the local app and would add attack surface and complexity. Tauri documents commands for frontend-to-Rust calls and events for lifecycle/state-change messaging, which maps exactly to Quorp’s runtime-event stream. ([Tauri][1])

---

## Why this fits Quorp extremely well

Quorp already has the hard Rust pieces:

1. **Runtime events are rich enough for a desktop timeline.** `RuntimeEvent` includes run start/finish, model request start/finish, phase changes, tool start/finish, validation, path-resolution failures, recovery events, parser recovery, verifier drain, policy denials, failed edits, checkpoints, and subscriber backpressure. The runtime already has named event subscribers, including renderer/proof/benchmark-style subscriptions. 

2. **The action protocol is already UI-friendly.** `AgentAction` is strongly typed, and `approval_policy()` distinguishes read-only auto-approved actions from mutating actions requiring explicit confirmation. That is exactly what the desktop permission UI needs. 

3. **Settings/trust/sandbox policy already exists.** Defaults are conservative: `TmpCopy` sandbox, `Ask` permissions, network/MCP/browser/process control disabled, and untrusted project settings cannot elevate permissions. Full-auto is downgraded unless it stays sandboxed and network-off. This is a huge advantage for a desktop full-auto product. 

4. **Patch/diff/proof primitives are already strong.** `quorp_patch_vm` has hash preconditions, preview IDs, high-risk patch gates, rollback tokens, stale-hash rejection, and tests for rollback behavior. That should become the backbone of the desktop diff and rollback UI. 

5. **The existing terminal renderer can inspire the desktop layout, but should not be embedded as-is.** `quorp_render` already has deterministic transcript/shell/session render primitives, but the desktop should render structured events directly rather than screenshotting or emulating terminal output.

# 5. How the app wires into Quorp

## MVP wiring: reuse current headless runner

The fastest usable app should wrap the existing headless run path. The CLI already does this from fullscreen mode: it creates a result directory, writes `objective.md`, builds `HeadlessRunOptions`, creates an event channel, and calls `agent_runner::run_headless_agent_with_progress`. 

For the desktop MVP:

```text
React composer
  ↓ invoke("session_start" / "session_send")
Tauri command
  ↓
quorp_desktop_core::SessionController
  ↓
quorp_session::agent_runner::run_headless_agent_with_progress
  ↓
TuiEvent / RuntimeEvent translation
  ↓
Tauri AppHandle.emit("quorp://event", DesktopEvent)
  ↓
React event listener
  ↓
Virtualized timeline + inspector updates
```

This gets you a working desktop quickly.

## Better 1–2 week wiring: first-class desktop event sink

The current `run_headless_agent_with_progress` path turns runtime events into terminal-ish `TuiEvent::Chat` messages. That is enough for MVP, but lossy for a polished desktop.

The better path:

```text
Agent runtime
  ↓ RuntimeEventFanout
DesktopEventSink subscribes as "desktop_renderer"
  ↓
Typed DesktopEvent stream
  ↓
Tauri event channel
  ↓
React timeline/inspector
```

Because `RuntimeEventFanout` already supports named subscribers and bounded queues, adding a desktop subscriber is natural.

```rust
// crates/quorp_desktop_core/src/events.rs

use std::path::PathBuf;

use quorp_agent_core::RuntimeEvent;
use quorp_core::{PermissionMode, RunMode, SandboxMode};
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct DesktopEvent {
    pub seq: u64,
    pub session_id: String,
    pub workspace_id: String,
    pub ts_unix_ms: i64,
    pub kind: DesktopEventKind,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum DesktopEventKind {
    Runtime {
        event: RuntimeEvent,
    },
    WorkspaceAdded {
        path: PathBuf,
        trusted: bool,
    },
    StatusSnapshot {
        snapshot: DesktopStatusSnapshot,
    },
    PermissionRequest {
        request: PermissionRequestDto,
    },
    PermissionDecision {
        decision: PermissionDecisionDto,
    },
    DiffUpdated {
        summary: DiffSummaryDto,
    },
    RunArtifact {
        role: String,
        path: PathBuf,
        sha256: Option<String>,
    },
    Error {
        message: String,
    },
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct DesktopStatusSnapshot {
    pub run_mode: RunMode,

    pub sandbox_mode: SandboxMode,
    pub model: String,
    pub provider: String,
    pub request_count: usize,
    pub billed_tokens: u64,
    pub context_budget_used: Option<u64>,
```

Create:

```rust
// crates/quorp_desktop_core/src/session.rs

use std::{
    collections::HashMap,
    path::PathBuf,
    sync::{Arc, Mutex},
};

use quorp_core::{PermissionMode, RunMode, SandboxMode};
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct StartSessionRequest {
    pub workspace: PathBuf,
    pub prompt: String,
    pub model: String,
    pub provider_base_url: Option<String>,
    pub run_mode: RunMode,
    pub permission_mode: PermissionMode,
    pub sandbox_mode: SandboxMode,
    pub max_steps: usize,
    pub max_seconds: Option<u64>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct StartSessionResponse {
    pub session_id: String,
    pub run_id: String,
    pub result_dir: PathBuf,
}

#[derive(Debug)]
pub struct DesktopSession {
    pub session_id: String,
    pub workspace: PathBuf,
    pub result_dir: PathBuf,
    pub run_mode: RunMode,
    pub permission_mode: PermissionMode,
    pub sandbox_mode: SandboxMode,
}

#[derive(Default)]
pub struct SessionRegistry {
    sessions: Mutex<HashMap<String, Arc<DesktopSession>>>,
}

impl SessionRegistry {
    pub fn insert(&self, session: DesktopSession) -> Arc<DesktopSession> {
        let session = Arc::new(session);
        self.sessions
            .lock()
            .expect("session registry")
            .insert(session.session_id.clone(), session.clone());
        session
    }

    pub fn get(&self, id: &str) -> Option<Arc<DesktopSession>> {
        self.sessions.lock().ok()?.get(id).cloned()
    }
}
```

```rust
use quorp_desktop_core::session::SessionRegistry;

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct WorkspaceRecord {
    pub id: String,
    pub path: PathBuf,
    pub display_name: String,
    pub trusted: bool,

#[derive(Default)]

    pub seq: AtomicU64,
    pub sessions: SessionRegistry,
    pub workspaces: Mutex<BTreeMap<String, WorkspaceRecord>>,

impl AppState {
    pub fn next_seq(&self) -> u64 {
        self.seq.fetch_add(1, Ordering::Relaxed) + 1
```

## `apps/desktop/src-tauri/src/commands.rs`

```rust
use std::{
    path::PathBuf,
    sync::mpsc,
    time::{SystemTime, UNIX_EPOCH},
};

use quorp_core::{PermissionMode, RunMode, SandboxMode};
use quorp_desktop_core::{
    events::{DesktopEvent, DesktopEventKind, DesktopStatusSnapshot},
    session::{DesktopSession, StartSessionRequest, StartSessionResponse},
};
use tauri::{AppHandle, Manager, State};

use crate::{events, state::{AppState, WorkspaceRecord}};

type CommandResult<T> = Result<T, String>;

fn now_unix_ms() -> i64 {
    SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .ok()
        .and_then(|d| i64::try_from(d.as_millis()).ok())
        .unwrap_or_default()
}

#[tauri::command]
pub async fn app_bootstrap(state: State<'_, AppState>) -> CommandResult<serde_json::Value> {
    let workspaces = state
        .workspaces
        .lock()
        .map_err(|_| "workspace registry lock poisoned".to_string())?
        .values()
        .cloned()
        .collect::<Vec<_>>();

    Ok(serde_json::json!({
        "workspaces": workspaces,
        "slash_commands": quorp_slash::Registry::new()
            .all()
            .iter()
            .map(|cmd| serde_json::json!({
                "name": cmd.name,
                "aliases": cmd.aliases,
                "description": cmd.description,
                "takes_args": cmd.takes_args,
            }))
            .collect::<Vec<_>>(),
    }))
}

#[tauri::command]
pub async fn workspace_add(
    app: AppHandle,
    state: State<'_, AppState>,
    path: PathBuf,
) -> CommandResult<WorkspaceRecord> {
    let canonical = std::fs::canonicalize(&path).unwrap_or(path.clone());
    let loaded = quorp_config::load_settings(&canonical)
        .map_err(|e| format!("failed to load workspace settings: {e:#}"))?;

    let id = loaded.trust.project_id.clone();
    let display_name = canonical
        .file_name()
        .and_then(|n| n.to_str())
        .unwrap_or("workspace")
        .to_string();

    let record = WorkspaceRecord {
        id: id.clone(),
        path: canonical.clone(),
        display_name,
        trusted: loaded.trust.trusted,
    };

    state
        .workspaces
        .lock()
        .map_err(|_| "workspace registry lock poisoned".to_string())?
        .insert(id.clone(), record.clone());

    events::emit(&app, DesktopEvent {
        seq: state.next_seq(),
        session_id: String::new(),
        workspace_id: id,
        ts_unix_ms: now_unix_ms(),
        kind: DesktopEventKind::WorkspaceAdded {
            path: canonical,
            trusted: record.trusted,
        },
    });

#[tauri::command]
pub async fn session_start(
    app: AppHandle,
    state: State<'_, AppState>,
    request: StartSessionRequest,
) -> CommandResult<StartSessionResponse> {
    enforce_desktop_safety(&request)?;

    let run_id = format!("desktop-{}", now_unix_ms());
    let result_dir = quorp_session::quorp::run_support::default_run_result_dir(
        &request.workspace,
        "desktop",
    );
    std::fs::create_dir_all(&result_dir)
        .map_err(|e| format!("failed to create result dir: {e}"))?;

    let objective_file = result_dir.join("objective.md");
    std::fs::write(&objective_file, &request.prompt)
        .map_err(|e| format!("failed to write objective: {e}"))?;

    let session_id = run_id.clone();

    let session = DesktopSession {
        session_id: session_id.clone(),
        workspace: request.workspace.clone(),
        result_dir: result_dir.clone(),
        run_mode: request.run_mode,
        permission_mode: request.permission_mode,
        sandbox_mode: request.sandbox_mode,
    };
    state.sessions.insert(session);

    let workspace_id = request.workspace.display().to_string();
    let (event_tx, event_rx) =
        mpsc::sync_channel::<quorp_session::quorp::tui::TuiEvent>(512);

    let autonomy_profile = match request.run_mode {
        RunMode::Plan => quorp_agent_core::AutonomyProfile::Interactive,
        RunMode::Act => {
            if request.sandbox_mode == SandboxMode::TmpCopy {
                quorp_agent_core::AutonomyProfile::AutonomousSandboxed
            } else {
                quorp_agent_core::AutonomyProfile::AutonomousHost
            }
        }
    };

    let options = quorp_session::quorp::agent_runner::HeadlessRunOptions {
        workspace: request.workspace.clone(),
        objective_file,
        model_id: request.model.clone(),
        base_url_override: request.provider_base_url.clone(),
        max_steps: request.max_steps,
        max_seconds: request.max_seconds,
        max_total_tokens: None,
        result_dir: result_dir.clone(),
        autonomy_profile,
        completion_policy: quorp_agent_core::CompletionPolicy::default(),
        objective_metadata: serde_json::json!({
            "origin": "desktop",
            "permission_mode": format!("{:?}", request.permission_mode),
            "sandbox": format!("{:?}", request.sandbox_mode),
        }),
        seed_context: Vec::new(),
    };

    let app_for_events = app.clone();
    let state_seq = app.state::<AppState>();
    let event_session_id = session_id.clone();
    let event_workspace_id = workspace_id.clone();

    std::thread::spawn(move || {
        while let Ok(event) = event_rx.recv() {
            let payload = serde_json::to_value(&event).unwrap_or(serde_json::Value::Null);
            events::emit(&app_for_events, DesktopEvent {
                seq: state_seq.next_seq(),
                session_id: event_session_id.clone(),
                workspace_id: event_workspace_id.clone(),
                ts_unix_ms: now_unix_ms(),
                kind: DesktopEventKind::RuntimeTextCompat { payload },
            });
        }
    });

    std::thread::spawn(move || {
        let outcome =
            quorp_session::quorp::agent_runner::run_headless_agent_with_progress(
                options,
                Some(event_tx),
            );

        if let Err(error) = outcome {
            log::error!("desktop agent run failed: {error:#}");
        }
    });

    Ok(StartSessionResponse {
        session_id,
        run_id,
        result_dir,
    })
}

fn enforce_desktop_safety(request: &StartSessionRequest) -> CommandResult<()> {
    if request.permission_mode == PermissionMode::FullAuto
        && request.sandbox_mode == SandboxMode::Host
    {
        return Err(
            "Full Auto requires tmp-copy or container sandbox from the desktop app".to_string(),
        );
    }

    if request.permission_mode == PermissionMode::FullPermissions {
        return Err(
            "Full Permissions must be enabled through an explicit trusted-project prompt"
                .to_string(),
        );
    }

    Ok(())
}
```

```ts
export const QUORP_EVENT_TOPIC = "quorp://event";

export interface DesktopEvent {
  seq: number;
  session_id: string;
  workspace_id: string;
  ts_unix_ms: number;
  kind: DesktopEventKind;
}

export type DesktopEventKind =
  | { type: "runtime"; event: unknown }
  | { type: "runtime_text_compat"; payload: unknown }
  | { type: "workspace_added"; path: string; trusted: boolean }
  | { type: "status_snapshot"; snapshot: unknown }
  | { type: "permission_request"; request: unknown }
  | { type: "permission_decision"; decision: unknown }
  | { type: "diff_updated"; summary: unknown }
  | { type: "run_artifact"; role: string; path: string; sha256?: string | null }
  | { type: "error"; message: string };

export async function listenForDesktopEvents(
  onEvent: (event: DesktopEvent) => void,
): Promise<UnlistenFn> {
  return listen<DesktopEvent>(QUORP_EVENT_TOPIC, (event) => {
    onEvent(event.payload);
  });
```

## `apps/desktop/src/app/store.ts`

```ts
import { create } from "zustand";
import type { DesktopEvent } from "../api/events";
import type { WorkspaceRecord } from "../api/ipc";

interface TimelineEvent {
  seq: number;
  sessionId: string;
  kind: DesktopEvent["kind"];
  ts: number;
}

interface AppStore {
  workspaces: WorkspaceRecord[];
  activeWorkspaceId?: string;
  activeSessionId?: string;
  timeline: TimelineEvent[];
  appendEvent: (event: DesktopEvent) => void;
  setWorkspaces: (workspaces: WorkspaceRecord[]) => void;
  setActiveWorkspace: (id: string) => void;
}

const MAX_IN_MEMORY_EVENTS = 3000;

export const useAppStore = create<AppStore>((set) => ({
  workspaces: [],
  timeline: [],

  appendEvent: (event) =>
    set((state) => {
      const timeline = [
        ...state.timeline,
        {
          seq: event.seq,
          sessionId: event.session_id,
          kind: event.kind,
          ts: event.ts_unix_ms,
        },
      ];

      return {
        timeline:
          timeline.length > MAX_IN_MEMORY_EVENTS
            ? timeline.slice(timeline.length - MAX_IN_MEMORY_EVENTS)
            : timeline,
      };
    }),

  setWorkspaces: (workspaces) => set({ workspaces }),
  setActiveWorkspace: (id) => set({ activeWorkspaceId: id }),
}));
```

# 11. Signed and notarized DMG pipeline later

Do not block the MVP on this, but design for it from day one.

Release pipeline:

```text
1. Build frontend
2. Compile Rust app
3. Bundle .app
4. Sign app with Developer ID Application cert
5. Create .dmg
6. Sign .dmg
7. Submit notarization
8. Staple ticket
9. Publish GitHub Release
10. Update latest.json for Tauri updater
```

Apple’s guidance for software distributed outside the Mac App Store is to sign with Developer ID and notarize; notarization is Apple’s automated malware/signing scan, and notarization tickets can be stapled to supported artifacts such as apps, DMGs, and packages. Apple also notes that `altool` stopped being accepted for notarization uploads after November 1, 2023, so the modern path is `notarytool` or Xcode’s organizer flow. ([Apple Help][7])

Tauri’s macOS signing docs say macOS code signing requires an Apple Developer account, a certificate installed locally or exported for CI, and notarization credentials; free Apple Developer accounts can be used for testing/development but cannot notarize. ([Tauri][6])

Later CI variables:

```bash
APPLE_SIGNING_IDENTITY="Developer ID Application: Your Name (TEAMID)"
APPLE_API_ISSUER="..."
APPLE_API_KEY="..."
APPLE_API_KEY_PATH="/path/to/AuthKey_XXXX.p8"
```

Build:

```bash
pnpm tauri build -- --bundles dmg
```

The Tauri updater can come after the first signed release. Its updater plugin requires signed update artifacts and can use a static JSON endpoint such as a GitHub Releases asset. ([Tauri][8])

# 14. The biggest traps to avoid

1. **Do not build the desktop app by spawning `quorp` as a subprocess forever.** It is okay for a one-day smoke test, but it will lose typed events, permissions, diffs, status, context pressure, and replay fidelity.

2. **Do not put React state in charge of truth.** Rust/session ledger is truth. React is a projection.

3. **Do not keep huge transcripts in frontend memory.** Keep a bounded event window in React and page older events from Rust/run artifacts.

4. **Do not expose shell execution through Tauri plugin permissions.** All execution should flow through Quorp’s existing action protocol, permission model, and sandbox.

5. **Do not let repo-authored settings silently elevate trust.** The existing config model already prevents untrusted project elevation; the desktop must preserve that visibly. 

6. **Do not let benchmark-specific prompt behavior leak into the desktop default.** `chat_service.rs` currently has a Qwen benchmark JSON contract branch. That should remain explicitly profile-gated, not a normal desktop-agent behavior.
