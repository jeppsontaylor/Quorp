# Recommendation

Build the desktop app as a new **Tauri 2 + React/TypeScript/Vite** app at:

```text
apps/desktop/
```

with the Tauri Rust crate at:

```text
apps/desktop/src-tauri/
```

and add it as a Cargo workspace member. Do **not** fold this into `crates/quorp`, `crates/quorp_cli`, `crates/quorp_term`, or `crates/quorp_render`.

Quorp is already shaped correctly for this: the repo describes itself as a terminal-first Rust agent runtime with `quorp_cli` split out and `quorp_session` carrying session/runtime helpers, headless autonomous runs, typed tools, sandbox/permission primitives, Patch VM writes, proof packets, memory/rules, and benchmark receipts. It also explicitly says release packaging/signing is not represented yet, so the desktop app should be a new app surface rather than another CLI refactor. ([GitHub][1])

The current binary is only a thin wrapper calling `quorp_cli::dispatch()`, so leave the CLI intact and add the desktop app as a sibling app that reuses the same runtime crates. ([GitHub][2])

# 2. Frontend stack

Use:

```text
Tauri 2
React
TypeScript
Vite
Zustand or useSyncExternalStore-based local store
TanStack Virtual for long event/diff lists
CodeMirror 6 only as a lazy-loaded editor/diff enhancement
xterm.js only for an optional terminal pane
Plain CSS/CSS variables, not a heavyweight component kit
```

Tauri is the right shell because it lets you bring a web frontend while keeping application logic in Rust, and it targets macOS, Windows, Linux, iOS, and Android from one codebase. ([Tauri][5]) Vite is a good fit because its build command outputs optimized static assets for production. ([vitejs][6]) TanStack Virtual is useful here because the agent event stream, tool output stream, and diff views can easily become huge; it is specifically built for virtualizing large JS/TS lists while retaining full markup/style control. ([TanStack][7])

Do **not** use WebSockets for the normal in-app bridge. Use:

```text
Frontend → Rust:
  Tauri invoke commands

Rust → Frontend:
  tauri::ipc::Channel for high-volume run streams
  Tauri events only for low-volume global notifications
```

Tauri docs say commands are the typed primitive for frontend-to-Rust calls, while events are dynamic; they also warn that events are not designed for low-latency/high-throughput streaming and recommend channels for streaming data. ([Tauri][8])

Use WebSockets only later for a **remote attach** mode, for example:

```text
quorp desktop-daemon --listen 127.0.0.1:46387 --auth-token <ephemeral>
```

That should not exist in v0.

# 5. Rust wiring

Use the existing runtime directly. Do **not** shell out to `quorp run` for the real app except as a temporary same-day smoke test.

Current `quorp_session::agent_runner` already wraps the runtime, records headless artifacts, maps `RuntimeEvent` into progress strings, writes model-turn artifacts, and has durable subscribers for memory/rule/proof/benchmark consumers. ([GitHub][13]) The desktop should add a typed sink beside that, not replace it.

## Desired runtime bridge

Add a desktop-friendly API in `crates/quorp_session`:

```rust
// crates/quorp_session/src/quorp/desktop_runtime.rs
use std::path::PathBuf;
use std::sync::{
    Arc,
    atomic::{AtomicBool, Ordering},
};

use anyhow::Context as _;
use quorp_agent_core::{AgentRunOutcome, RuntimeEventSink};

#[derive(Debug, Clone)]
pub struct DesktopRunOptions {
    pub workspace_root: PathBuf,
    pub cwd: PathBuf,
    pub objective: String,
    pub model_id: String,
    pub provider: Option<String>,
    pub permission_profile: String,
    pub sandbox_profile: String,
    pub max_iterations: usize,
    pub result_dir: PathBuf,
    pub enable_rollback_on_validation_failure: bool,
}

pub struct DesktopRunHandle {
    pub run_id: String,
    pub cancel: Arc<AtomicBool>,
}

pub async fn run_desktop_agent_with_sink(
    options: DesktopRunOptions,
    sink: Arc<dyn RuntimeEventSink>,
    cancellation_flag: Arc<AtomicBool>,
) -> anyhow::Result<AgentRunOutcome> {
    // Build the same request currently built by run_headless_agent_with_progress,
    // but pass the desktop sink into RuntimeEventFanout::with_downstream(...)
    // and set AgentRunRequest.cancellation_flag = Some(cancellation_flag).
    //
    // This should preserve:
    // - checkpoint.json
    // - events.jsonl
    // - summary.json
    // - transcript.json
    // - proof receipts
    // while also streaming typed events to the desktop.
    crate::quorp::agent_runner::run_headless_agent_with_typed_sink(
        options,
        sink,
        Some(cancellation_flag),
    )
    .await
    .context("desktop agent run failed")
}
```

Then add the actual API by refactoring today’s `run_headless_agent_with_progress` into:

```rust
run_headless_agent_common(options, progress_sink, typed_sink, cancellation_flag)
```

Acceptance criteria:

```text
- Existing CLI/headless behavior unchanged.
- Existing benchmark artifacts unchanged.
- Desktop receives typed RuntimeEvent values.
- cancellation_flag stops active runs.
- checkpoint.json still written.
- events.jsonl still written.
- proof-receipt.json still written.
```

# 8. Tauri command bridge

Use `tauri::ipc::Channel` for runtime streams.

```rust
// apps/desktop/src-tauri/src/commands.rs
use std::sync::{
    Arc,
    atomic::{AtomicBool, Ordering},
};

use quorp_agent_core::RuntimeEventSink;
use quorp_app_model::{DesktopRuntimeEvent, StartRunRequest};
use tauri::{ipc::Channel, State};

use crate::app_state::{DesktopRunRecord, DesktopState};
use crate::desktop_error::DesktopError;
use crate::runtime_bridge::{ChannelRuntimeSink, next_run_id};
use crate::workspace::resolve_workspace;

#[tauri::command(rename_all = "camelCase")]
pub async fn start_agent_run(
    state: State<'_, DesktopState>,
    req: StartRunRequest,
    stream: Channel<DesktopRuntimeEvent>,
) -> Result<String, DesktopError> {
    let workspace = resolve_workspace(&state, &req.workspace_id)?;
    let run_id = next_run_id();
    let cancel = Arc::new(AtomicBool::new(false));

    let sink: Arc<dyn RuntimeEventSink> = Arc::new(ChannelRuntimeSink::new(
        run_id.clone(),
        stream,
    ));

    let options = crate::runtime_bridge::build_desktop_run_options(
        &workspace,
        &run_id,
        req,
    )?;

    let cancel_for_task = Arc::clone(&cancel);
    let run_id_for_task = run_id.clone();

    let handle = tauri::async_runtime::spawn(async move {
        let outcome = quorp_session::quorp::desktop_runtime::run_desktop_agent_with_sink(
            options,
            sink,
            cancel_for_task,
        )
        .await;

        match outcome {
            Ok(outcome) => {
                log::info!("desktop run {run_id_for_task} finished: {:?}", outcome.stop_reason);
            }
            Err(error) => {
                log::error!("desktop run {run_id_for_task} failed: {error}");
            }
        }
    });

    state.insert_run(DesktopRunRecord {
        run_id: run_id.clone(),
        cancel,
        handle,
    })?;

    Ok(run_id)
}

#[tauri::command(rename_all = "camelCase")]
pub async fn cancel_agent_run(
    state: State<'_, DesktopState>,
    run_id: String,
) -> Result<(), DesktopError> {
    let run = state.get_run(&run_id)?;
    run.cancel.store(true, Ordering::Relaxed);
    Ok(())
}
```

```rust
// apps/desktop/src-tauri/src/runtime_bridge.rs
use std::sync::atomic::{AtomicU64, Ordering};

use quorp_agent_core::{RuntimeEvent, RuntimeEventSink};
use quorp_app_model::DesktopRuntimeEvent;
use tauri::ipc::Channel;

static NEXT_EVENT_SEQUENCE: AtomicU64 = AtomicU64::new(1);

pub struct ChannelRuntimeSink {
    run_id: String,
    tx: Channel<DesktopRuntimeEvent>,
}

impl ChannelRuntimeSink {
    pub fn new(run_id: String, tx: Channel<DesktopRuntimeEvent>) -> Self {
        Self { run_id, tx }
    }
}

impl RuntimeEventSink for ChannelRuntimeSink {
    fn emit(&self, event: RuntimeEvent) {
        let sequence = NEXT_EVENT_SEQUENCE.fetch_add(1, Ordering::Relaxed);

        let payload = match serde_json::to_value(&event) {
            Ok(value) => value,
            Err(error) => {
                log::error!("failed to serialize runtime event: {error}");
                return;
            }
        };

        let message = DesktopRuntimeEvent {
            run_id: self.run_id.clone(),
            sequence,
            kind: runtime_event_kind(&event).to_string(),
            payload,
        };

        if let Err(error) = self.tx.send(message) {
            log::warn!("desktop runtime stream closed: {error}");
        }
    }
}

fn runtime_event_kind(event: &RuntimeEvent) -> &'static str {
    match event {
        RuntimeEvent::RunStarted { .. } => "runStarted",
        RuntimeEvent::StatusUpdate { .. } => "statusUpdate",
        RuntimeEvent::PhaseChanged { .. } => "phaseChanged",
        RuntimeEvent::ModelRequestStarted { .. } => "modelRequestStarted",
        RuntimeEvent::ModelRequestFinished { .. } => "modelRequestFinished",
        RuntimeEvent::ToolCallStarted { .. } => "toolCallStarted",
        RuntimeEvent::ToolCallFinished { .. } => "toolCallFinished",
        RuntimeEvent::ValidationStarted { .. } => "validationStarted",
        RuntimeEvent::ValidationFinished { .. } => "validationFinished",
        RuntimeEvent::AssistantTurnSummary { .. } => "assistantTurnSummary",
        RuntimeEvent::CheckpointSaved { .. } => "checkpointSaved",
        RuntimeEvent::RunFinished { .. } => "runFinished",
        RuntimeEvent::FatalError { .. } => "fatalError",
        RuntimeEvent::SubscriberBackpressure { .. } => "subscriberBackpressure",
        RuntimeEvent::FailedEditRecorded { .. } => "failedEditRecorded",
        _ => "runtimeEvent",
    }
}
```

# 9. Workspace path security

Do not grant the frontend broad filesystem permissions. The frontend gets folder paths through the dialog plugin, but all reads/writes should go through Rust commands that check workspace IDs and canonical paths.

```rust
// apps/desktop/src-tauri/src/workspace.rs
use std::path::{Component, Path, PathBuf};

use crate::desktop_error::DesktopError;

pub fn resolve_child_path(root: &Path, relative: &str) -> Result<PathBuf, DesktopError> {
    let rel = Path::new(relative);

    if rel.is_absolute() {
        return Err(DesktopError::PathEscapesWorkspace);
    }

    if rel.components().any(|part| matches!(part, Component::ParentDir)) {
        return Err(DesktopError::PathEscapesWorkspace);
    }

    let canonical_root = dunce::canonicalize(root)?;
    let candidate = canonical_root.join(rel);

    let canonical_candidate = match dunce::canonicalize(&candidate) {
        Ok(path) => path,
        Err(_) => candidate,
    };

    if !canonical_candidate.starts_with(&canonical_root) {
        return Err(DesktopError::PathEscapesWorkspace);
    }

    Ok(canonical_candidate)
}
```

Tauri’s v2 security model uses capabilities to constrain which permissions are granted to which windows/webviews, so keep capabilities minimal and let Quorp’s own permission/sandbox system mediate dangerous actions. ([Tauri][15])

```ts
import { invoke, Channel } from "@tauri-apps/api/core";
import { open } from "@tauri-apps/plugin-dialog";
import { useAppStore } from "../app/store";

export type DesktopRuntimeEvent = {
  runId: string;
  sequence: number;
  kind: string;
  payload: unknown;
};

export type StartRunRequest = {
  workspaceId: string;
  prompt: string;
  mode: "ask" | "plan" | "act" | "full-auto";
  permissionProfile:
    | "ask"
    | "auto-safe"
    | "accept-edits"
    | "yolo-sandbox"
    | "full-permissions-host";
  sandboxProfile:
    | "tmp-copy"
    | "git-worktree"
    | "host-read-only"
    | "host-full";
  modelId?: string;
};

export async function addFolder(): Promise<void> {
  const selected = await open({
    directory: true,
    multiple: true,
    title: "Add folder to Quorp",
  });

  if (!selected) return;

  const paths = Array.isArray(selected) ? selected : [selected];
  await invoke("add_workspace", { paths });
  await refreshWorkspaces();
}

export async function refreshWorkspaces(): Promise<void> {
  const workspaces = await invoke("list_workspaces");
  useAppStore.getState().setWorkspaces(workspaces as never);
}

export async function startRun(req: StartRunRequest): Promise<string> {
  const stream = new Channel<DesktopRuntimeEvent>();

  stream.onmessage = (event) => {
    useAppStore.getState().ingestRuntimeEvent(event);
  };

  const runId = await invoke<string>("start_agent_run", {
    req,
    stream,
  });

  useAppStore.getState().markRunStarted(runId, req);
  return runId;
}

export async function cancelRun(runId: string): Promise<void> {
  await invoke("cancel_agent_run", { runId });
}
```

# 13. Virtual event stream

Do not keep 50,000 DOM nodes alive. Keep normalized event state and virtualize.

```tsx
// apps/desktop/src/features/stream/AgentStream.tsx
import { useRef } from "react";
import { useVirtualizer } from "@tanstack/react-virtual";
import { useAppStore } from "../../app/store";
import { RuntimeEventCard } from "./RuntimeEventCard";

export function AgentStream() {
  const parentRef = useRef<HTMLDivElement | null>(null);
  const events = useAppStore((state) => state.activeRunEvents);

  const virtualizer = useVirtualizer({
    count: events.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 84,
    overscan: 12,
  });

  return (
    <div ref={parentRef} className="agent-stream">
      <div
        style={{
          height: virtualizer.getTotalSize(),
          position: "relative",
        }}
      >
        {virtualizer.getVirtualItems().map((row) => {
          const event = events[row.index];

          return (
            <div
              key={event.sequence}
              className="virtual-row"
              style={{
                position: "absolute",
                top: 0,
                left: 0,
                right: 0,
                transform: `translateY(${row.start}px)`,
              }}
            >
              <RuntimeEventCard event={event} />
            </div>
          );
        })}
      </div>
    </div>
  );
}
```

# 15. Permissions UX

Default for newly added folders:

```text
trusted: false
permission: Ask
sandbox: tmp-copy or git-worktree
network: off unless needed
MCP: off unless enabled
browser: off unless enabled
dependency installs: ask
host full permissions: disabled
```

Quorp already supports sandbox modes `host`, `tmp-copy`, and `git-worktree` with fallback, permission primitives such as `read-only`, `ask`, `accept-edits`, `auto-safe`, and `yolo-sandbox`, and explicitly rejects `--yolo` with direct host sandboxing. ([GitHub][1]) Preserve that in the desktop UX.

The permission picker should make danger visible:

```text
Ask
  Safest. Prompts for writes, commands, network, MCP, browser.

Auto-safe
  Allows read-only and known-safe validation. Prompts for mutation.

Accept edits
  Allows Patch VM edits. Prompts for shell/network/dependency installs.

Yolo sandbox
  Allows full-auto only inside tmp-copy/git-worktree sandbox.

Full permissions on host
  Red. Requires typed confirmation. Never default. Never enabled for untrusted folders.
```

# 22. Key implementation warnings

1. **Do not duplicate CLI logic.** Desktop should use `quorp_session` and `quorp_agent_core`, not copy `quorp_cli` internals.

2. **Do not stream everything into React state.** Keep big artifacts in Rust/filesystem and fetch chunks.

3. **Do not grant broad Tauri fs/shell permissions.** Use typed Rust commands and Quorp’s existing security model.

4. **Do not make host full-auto convenient.** It should be possible, visible, logged, and gated, but never default.

5. **Repair macOS PATH early.** Otherwise `cargo`, `git`, `rg`, and `fd` may work in the terminal but fail inside the `.app`.

6. **Keep CLI and desktop releases separate.** CLI remains `quorp`; desktop binary is `quorp-desktop` inside `Quorp.app`.

7. **Make every run replayable.** Desktop should preserve `request.json`, `events.jsonl`, `summary.json`, `transcript.json`, `final.diff`, `proof-receipt.json`, and `checkpoint.json`.
