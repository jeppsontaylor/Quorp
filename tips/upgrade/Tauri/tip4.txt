## Recommendation

Build the desktop app as a **Tauri 2 app under `apps/desktop`**, keep the terminal CLI as the existing `quorp` binary, and have both surfaces call the same Rust runtime/session crates.

Do **not** make the desktop app a shell around `quorp run` subprocesses. That would waste memory, make cancellation/streaming harder, and lose typed runtime state. The right design is:

```text
apps/desktop
  React + TypeScript + Vite UI
  Tauri 2 macOS .app/.dmg shell
        ↓
crates/quorp_desktop       new thin Rust app facade
        ↓
crates/quorp_session       existing headless/session runner
crates/quorp_agent_core    existing runtime loop/events/checkpoints
crates/quorp_agent_protocol existing wire envelopes
crates/quorp_permissions   existing permission policy
crates/quorp_sandbox       existing sandbox creation
crates/quorp_core          shared modes/settings/proof types
```

The fit is strong because Quorp already has a single terminal binary, extracted `quorp_cli` and `quorp_session` crates, a headless autonomous run path, typed runtime events, Patch VM write receipts, proof packets, sandboxing, permissions, memory/rule scaffolding, and benchmark artifacts. The repo support doc explicitly describes that current shape, and the binary itself is already thin: `crates/quorp/src/main.rs` delegates to `quorp_cli::dispatch()`. ([GitHub][1])

One caveat: I could not perform a local clone/build in this sandbox because DNS resolution for GitHub failed. The plan below is based on the live GitHub repo/raw files and official Tauri/Apple docs, not a local `cargo check`.

Why this split:

```text
apps/desktop
```

is the actual product app: web assets, Tauri config, macOS bundle config, icons, capabilities, DMG config.

```text
crates/quorp_desktop
```

is the Rust facade that is testable without launching a WebView. It owns workspace registry, desktop run manager, request/response DTOs, event batching, and artifact lookup. The Tauri crate should stay thin.

Do **not** put this into `crates/quorp` or `crates/quorp_cli`. `crates/quorp` is already the terminal binary wrapper, and `quorp_cli` owns terminal command dispatch. The desktop app is a sibling product surface, not a CLI mode.

The workspace already has the right crate style: many dedicated `quorp_*` crates, workspace dependencies, and `[lib] path = ...` patterns. The root workspace includes `quorp_cli`, `quorp_session`, `quorp_agent_core`, `quorp_agent_protocol`, `quorp_permissions`, `quorp_sandbox`, `quorp_render`, `quorp_slash`, `quorp_term`, and related crates. ([GitHub][2])

## 2. Frontend technology stack

Use this stack:

```text
Tauri 2
React
TypeScript
Vite
Zustand or Jotai for small normalized stores
TanStack Virtual for transcript/tool/diff virtualization
CodeMirror 6 only for file preview / diff details, dynamically loaded
Plain CSS variables, no heavy component library
Tauri commands + channels/events, not WebSockets
```

### Why not WebSockets?

For a local Tauri app, WebSockets are the wrong default. They add a local server, port management, auth questions, and another transport to secure. Tauri already has Rust-to-frontend communication: frontend-to-Rust commands, Rust-to-frontend events, and channels. Tauri documents commands as the primitive for calling Rust from the web app, and Rust can call the frontend through the event system or channels; the docs also note that events are for smaller streams and channels are the streaming-oriented path. ([Tauri][3])

Use:

```text
Tauri command/invoke:
  add_workspace
  start_agent
  stop_agent
  resume_run
  list_recent_runs
  read_artifact
  open_settings
  set_permission_mode
  set_sandbox_mode
  compact_context
  verify_now
  rollback_to_checkpoint

Tauri Channel:
  high-volume runtime event stream
  command output chunks
  diff chunks

Tauri events:
  low-volume global notifications
  settings changed
  workspace added
  run completed
```

### Why React/TypeScript?

Quorp is Rust-first, but a polished Codex/Claude-style desktop UI needs excellent layout, virtualized scrollback, command palettes, diff rendering, forms, menus, keyboard shortcuts, theme switching, and fast iteration. Tauri is designed for exactly this split: web frontend plus Rust backend, and it supports arbitrary frontend stacks while keeping the backend as Rust. ([Tauri][4])

A pure Rust UI with egui/Slint is possible, but it will slow down the polish path for this specific product. For a high-density agent IDE with diff cards, sidebars, inspectors, settings, animations, and resizable panes, **Tauri + React/TypeScript is the fastest serious path**.

### Memory-conscious rules

The UI should never keep the whole run in React state.

Use this model:

```text
Disk:
  full events.jsonl
  transcript.json
  summary.json
  metadata.json
  final.diff
  proof-receipt.json
  logs/

Rust backend:
  active run state
  bounded event ring buffer
  artifact index
  event batching
  cancellation handles

Frontend:
  normalized current run metadata
  visible virtualized timeline rows
  collapsed event summaries
  selected artifact/diff window
  recent N command-output chunks only
```

The existing headless run path already writes `request.json`, `transcript.json`, `summary.json`, `metadata.json`, and `final.diff`, and records event/usage data through the headless recorder. The desktop should reuse that artifact model instead of inventing a separate persistence layer. ([GitHub][5])

## 5. Runtime wiring

### Existing runtime path to reuse

The current CLI path already creates provider/model/base URL/sandbox config, creates result directories, invokes `run_headless_agent`, writes artifacts, copies logs, and supports resume from `request.json` and `checkpoint.json`. ([GitHub][8])

For the desktop, use the same path but expose it through a desktop run manager.

```text
React UI
  invoke("start_agent")
        ↓
Tauri command
        ↓
quorp_desktop::RunManager
        ↓
quorp_session::agent_runner
        ↓
quorp_agent_core::run_agent_task
        ↓
RuntimeEventSink / RuntimeEventFanout
        ↓
Tauri Channel → React timeline
```

### Important change to `quorp_session`

Right now, `run_headless_agent_with_progress` is close to what the desktop needs, but the app should receive **structured `RuntimeEvent`**, not only summarized TUI text events.

Add a new API in:

```text
crates/quorp_session/src/quorp/agent_runner.rs
```

Proposed API:

```rust
use std::sync::Arc;

use quorp_agent_core::runtime::RuntimeEventSink;

pub struct HeadlessRunHooks {
    pub runtime_event_sink: Option<Arc<dyn RuntimeEventSink>>,
}

impl Default for HeadlessRunHooks {
    fn default() -> Self {
        Self {
            runtime_event_sink: None,
        }
    }
}

pub fn run_headless_agent_with_hooks(
    options: HeadlessRunOptions,
    progress_tx: Option<std::sync::mpsc::SyncSender<TuiEvent>>,
    hooks: HeadlessRunHooks,
) -> anyhow::Result<HeadlessRunResult> {
    // Existing run_headless_agent_with_progress body moves here.
    //
    // Construct RuntimeEventFanout with the optional downstream sink:
    //
    // let event_fanout = if let Some(sink) = hooks.runtime_event_sink {
    //     RuntimeEventFanout::with_downstream(sink)
    // } else {
    //     RuntimeEventFanout::new()
    // };
    //
    // Existing durable subscribers continue to subscribe to event_fanout:
    // memory_writer, rule_forge_observer, proof_recorder, benchmark_recorder.
    //
    // Existing progress sink can stay as a renderer subscriber or separate sink.
}

pub fn run_headless_agent_with_progress(
    options: HeadlessRunOptions,
    progress_tx: Option<std::sync::mpsc::SyncSender<TuiEvent>>,
) -> anyhow::Result<HeadlessRunResult> {
    run_headless_agent_with_hooks(options, progress_tx, HeadlessRunHooks::default())
}
```

This keeps CLI behavior stable and gives the desktop a structured event stream.

Quorp already has `RuntimeEventSink`, `RuntimeEventFanout`, named subscribers, renderer subscription, backpressure events, and durable workers for memory/rule/proof/benchmark consumers. The desktop should be another downstream sink/subscriber, not a replacement. ([GitHub][5])

```rust
use std::{
    collections::BTreeMap,
    path::{Path, PathBuf},
    sync::{Arc, Mutex},
};

use anyhow::{Context, Result};
use quorp_core::{PermissionMode, SandboxMode};
use serde::{Deserialize, Serialize};
use uuid::Uuid;

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct WorkspaceInfo {
    pub id: String,
    pub root: PathBuf,
    pub display_name: String,
    pub trusted: bool,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct StartAgentRequest {
    pub workspace_id: String,
    pub prompt: String,
    pub model_id: Option<String>,
    pub base_url: Option<String>,
    pub permission_mode: PermissionMode,
    pub sandbox_mode: SandboxMode,
    pub max_steps: Option<usize>,
    pub max_seconds: Option<u64>,
    pub max_total_tokens: Option<u64>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RunHandle {
    pub run_id: String,
    pub workspace_id: String,
    pub result_dir: PathBuf,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum DesktopEvent {
    RunStarted {
        run_id: String,
        result_dir: PathBuf,
    },
    Runtime {
        run_id: String,
        event: serde_json::Value,
    },
    ArtifactWritten {
        run_id: String,
        kind: String,
        path: PathBuf,
    },
    RunFinished {
        run_id: String,
        summary_path: PathBuf,
    },
    RunFailed {
        run_id: String,
        message: String,
    },
}

#[derive(Default)]
pub struct WorkspaceRegistry {
    workspaces: Mutex<BTreeMap<String, WorkspaceInfo>>,
}

impl WorkspaceRegistry {
    pub fn add_workspace(&self, root: PathBuf) -> Result<WorkspaceInfo> {
        let canonical_root = root
            .canonicalize()
            .with_context(|| format!("failed to canonicalize {}", root.display()))?;

        let display_name = canonical_root
            .file_name()
            .and_then(|name| name.to_str())
            .unwrap_or("workspace")
            .to_owned();

        let workspace = WorkspaceInfo {
            id: Uuid::new_v4().to_string(),
            root: canonical_root,
            display_name,
            trusted: false,
        };

        let mut workspaces = self
            .workspaces
            .lock()
            .map_err(|_| anyhow::anyhow!("workspace registry lock poisoned"))?;

        workspaces.insert(workspace.id.clone(), workspace.clone());
        Ok(workspace)
    }

    pub fn get(&self, id: &str) -> Result<WorkspaceInfo> {
        let workspaces = self
            .workspaces
            .lock()
            .map_err(|_| anyhow::anyhow!("workspace registry lock poisoned"))?;

        workspaces
            .get(id)
            .cloned()
            .with_context(|| format!("unknown workspace id `{id}`"))
    }

    pub fn list(&self) -> Result<Vec<WorkspaceInfo>> {
        let workspaces = self
            .workspaces
            .lock()
            .map_err(|_| anyhow::anyhow!("workspace registry lock poisoned"))?;

        Ok(workspaces.values().cloned().collect())
    }
}

pub struct DesktopState {
    pub workspaces: Arc<WorkspaceRegistry>,
    pub app_data_dir: PathBuf,
}

impl DesktopState {
    pub fn new(app_data_dir: PathBuf) -> Self {
        Self {
            workspaces: Arc::new(WorkspaceRegistry::default()),
            app_data_dir,
        }
    }
}
```

### `apps/desktop/src-tauri/src/commands.rs`

This is the core bridge. For the first MVP, stream summarized events; after the `run_headless_agent_with_hooks` patch, stream structured runtime events.

```rust
use std::{path::PathBuf, sync::Arc, thread};

use quorp_agent_core::runtime::{RuntimeEvent, RuntimeEventSink};
use quorp_desktop::{
    DesktopEvent, DesktopState, RunHandle, StartAgentRequest, WorkspaceInfo,
};
use serde::Serialize;
use tauri::{ipc::Channel, Manager};

#[derive(Debug, Serialize)]
pub struct CommandError {
    pub message: String,
}

impl From<anyhow::Error> for CommandError {
    fn from(error: anyhow::Error) -> Self {
        Self {
            message: format!("{error:#}"),
        }
    }
}

#[tauri::command]
pub async fn add_workspace(
    state: tauri::State<'_, DesktopState>,
    path: PathBuf,
) -> Result<WorkspaceInfo, CommandError> {
    state.workspaces.add_workspace(path).map_err(Into::into)
}

#[tauri::command]
pub async fn list_workspaces(
    state: tauri::State<'_, DesktopState>,
) -> Result<Vec<WorkspaceInfo>, CommandError> {
    state.workspaces.list().map_err(Into::into)
}

#[tauri::command]
pub async fn start_agent(
    state: tauri::State<'_, DesktopState>,
    request: StartAgentRequest,
    on_event: Channel<DesktopEvent>,
) -> Result<RunHandle, CommandError> {
    let workspace = state.workspaces.get(&request.workspace_id)?;

    let run_id = uuid::Uuid::new_v4().to_string();
    let result_dir = workspace
        .root
        .join(".quorp-runs")
        .join("desktop")
        .join(&run_id);

    std::fs::create_dir_all(&result_dir)
        .map_err(|error| anyhow::anyhow!("failed to create run dir: {error}"))?;

    let handle = RunHandle {
        run_id: run_id.clone(),
        workspace_id: workspace.id.clone(),
        result_dir: result_dir.clone(),
    };

    on_event
        .send(DesktopEvent::RunStarted {
            run_id: run_id.clone(),
            result_dir: result_dir.clone(),
        })
        .map_err(|error| anyhow::anyhow!("failed to send run-started event: {error}"))?;

    let sink = Arc::new(DesktopRuntimeSink {
        run_id: run_id.clone(),
        on_event: on_event.clone(),
    });

    thread::spawn(move || {
        let outcome = run_quorp_agent(workspace.root, result_dir.clone(), request, sink);

        match outcome {
            Ok(summary_path) => {
                if let Err(error) = on_event.send(DesktopEvent::RunFinished {
                    run_id,
                    summary_path,
                }) {
                    log::error!("failed to send run-finished desktop event: {error}");
                }
            }
            Err(error) => {
                if let Err(send_error) = on_event.send(DesktopEvent::RunFailed {
                    run_id,
                    message: format!("{error:#}"),
                }) {
                    log::error!("failed to send run-failed desktop event: {send_error}");
                }
            }
        }
    });

    Ok(handle)
}

struct DesktopRuntimeSink {
    run_id: String,
    on_event: Channel<DesktopEvent>,
}

impl RuntimeEventSink for DesktopRuntimeSink {
    fn emit(&self, event: RuntimeEvent) {
        let value = match serde_json::to_value(event) {
            Ok(value) => value,
            Err(error) => {
                log::error!("failed to serialize runtime event for desktop: {error}");
                return;
            }
        };

        if let Err(error) = self.on_event.send(DesktopEvent::Runtime {
            run_id: self.run_id.clone(),
            event: value,
        }) {
            log::error!("failed to send runtime event to desktop: {error}");
        }
    }
}

fn run_quorp_agent(
    workspace_root: PathBuf,
    result_dir: PathBuf,
    request: StartAgentRequest,
    runtime_sink: Arc<dyn RuntimeEventSink>,
) -> anyhow::Result<PathBuf> {
    use quorp_session::quorp::agent_runner::{
        run_headless_agent_with_hooks, HeadlessRunHooks, HeadlessRunOptions,
    };

    let options = HeadlessRunOptions {
        workspace: workspace_root.clone(),
        objective: request.prompt,
        model_id: request.model_id,
        base_url: request.base_url,
        max_steps: request.max_steps,
        max_seconds: request.max_seconds,
        max_total_tokens: request.max_total_tokens,
        result_dir: result_dir.clone(),
        // Fill remaining fields from existing defaults/helper constructors.
        ..HeadlessRunOptions::for_workspace(workspace_root)
    };

    let hooks = HeadlessRunHooks {
        runtime_event_sink: Some(runtime_sink),
    };

    run_headless_agent_with_hooks(options, None, hooks)?;

    Ok(result_dir.join("summary.json"))
}

#[tauri::command]
pub async fn stop_agent(_run_id: String) -> Result<(), CommandError> {
    // Implement with cancellation_flag storage in RunManager.
    Ok(())
}

#[tauri::command]

    std::fs::read_to_string(&path)
        .map_err(|error| anyhow::anyhow!("failed to read {}: {error}", path.display()).into())
```

### Frontend store

```ts
// apps/desktop/src/stores/runStore.ts
import { create } from "zustand";
import type { DesktopEvent } from "../tauri/commands";

export interface TimelineRow {
  id: string;
  runId: string;
  kind: string;
  title: string;
  payload: unknown;
  createdAt: number;
}

interface RunState {
  activeRunId?: string;
  rows: TimelineRow[];
  tokenTotal: number;
  status: string;
  applyDesktopEvent: (event: DesktopEvent) => void;
}

export const useRunStore = create<RunState>((set) => ({
  rows: [],
  tokenTotal: 0,
  status: "idle",

  applyDesktopEvent(event) {
    set((state) => {
      if (event.type === "run_started") {
        return {
          ...state,
          activeRunId: event.run_id,
          status: "running",
          rows: [
            ...state.rows,
            {
              id: `${event.run_id}:started`,
              runId: event.run_id,
              kind: "run_started",
              title: "Run started",
              payload: event,
              createdAt: Date.now(),
            },
          ],
        };
      }

      if (event.type === "runtime") {
        const runtimeEvent = event.event as { event?: string };

        return {
          ...state,
          rows: [
            ...state.rows,
            {
              id: `${event.run_id}:${state.rows.length}`,
              runId: event.run_id,
              kind: runtimeEvent.event ?? "runtime",
              title: runtimeEvent.event ?? "Runtime event",
              payload: event.event,
              createdAt: Date.now(),
            },
          ],
        };
      }

      if (event.type === "run_finished") {
        return {
          ...state,
          status: "finished",
          rows: [
            ...state.rows,
            {
              id: `${event.run_id}:finished`,
              runId: event.run_id,
              kind: "run_finished",
              title: "Run finished",
              payload: event,
              createdAt: Date.now(),
            },
          ],
        };
      }

      if (event.type === "run_failed") {
        return {
          ...state,
          status: "failed",
          rows: [
            ...state.rows,
            {
              id: `${event.run_id}:failed`,
              runId: event.run_id,
              kind: "run_failed",
              title: "Run failed",
              payload: event,
              createdAt: Date.now(),
            },
          ],
        };
      }

      return state;
    });
  },
}));
```

```tsx
// apps/desktop/src/components/AgentTimeline.tsx
import { useRef } from "react";

import { useRunStore } from "../stores/runStore";
```

## 7. Permission and sandbox UX

Default for newly added folders:

```text
trusted: false
permission: ask
sandbox: tmp-copy
network: off unless explicitly requested
MCP: off unless configured
browser: off unless explicitly enabled
generated executables: prompt
dependency installs: prompt
```

For trusted folders:

```text
permission: auto-safe
sandbox: tmp-copy or host, user choice
network: prompt
dependency installs: prompt
```

For full-auto:

```text
permission: yolo-sandbox
sandbox: tmp-copy or container only
host: forbidden
```

This maps to Quorp’s existing sandbox backends: host, tmp-copy, and git-worktree with tmp-copy fallback. The sandbox crate excludes heavy/unwanted paths for tmp copy, has environment allow/deny policy, and has a container command plan with `--network=none`, `--cap-drop=ALL`, `no-new-privileges`, bind mount, and user mapping. ([GitHub][11])

The permission picker should be visible in the composer and status ribbon:

```text
[Ask] [Auto-safe] [Full permissions] [Yolo sandbox]
```

When the user chooses a dangerous combination:

```text
Full Permissions + Host
```

show a modal:

```text
This gives the agent direct host write/run access in this workspace.

Risks:
  - destructive shell commands
  - dependency install side effects
  - generated executable execution
  - secret reads
  - git remote mutation

Recommended: Auto-safe + tmp-copy.

[Cancel] [Use Auto-safe tmp-copy] [I understand: Full Host]
```

For public-quality UX, every mutating action should show:

```text
who requested it
tool/action name
path/command
permission decision
sandbox path
hash/precondition if write
receipt/proof link
rollback availability
```

## 9. DMG build path

### Quick local testing on your Mac

This should be the first goal: run the desktop app and build a local DMG without waiting on Developer ID notarization.

```bash
# from repo root
cd apps/desktop

pnpm install

# fast app dev
pnpm tauri dev

# local production app + dmg
pnpm tauri build --bundles app,dmg
```

Expected output will usually be under something like:

```text
apps/desktop/src-tauri/target/release/bundle/macos/Quorp.app
apps/desktop/src-tauri/target/release/bundle/dmg/Quorp_*.dmg
```

If the workspace sets a shared `CARGO_TARGET_DIR`, the target path may be under the shared target directory instead.

For local testing, use ad-hoc signing:

```json
{
  "bundle": {
    "macOS": {
      "signingIdentity": "-"
    }
  }
}
```

This is not the same as public distribution. It is good enough to avoid blocking the first usable Mac build. Tauri’s docs explicitly describe ad-hoc signing with `signingIdentity: "-"` and note that it can still require user whitelisting in macOS Privacy & Security. ([Tauri][10])

### Real release build

Tauri can bundle an app and DMG for macOS distribution; the docs show `tauri bundle --bundles app,dmg` / `cargo tauri bundle --bundles app,dmg`, and state that macOS direct-download distribution uses a DMG and requires signing/notarization for distribution outside the App Store. ([Tauri][13])

Release pipeline:

```text
1. Build frontend
2. Compile Rust app
3. Bundle .app
4. Sign .app with Developer ID Application certificate
5. Create .dmg
6. Sign .dmg
7. Submit for notarization
8. Staple notarization ticket
9. Publish release
```

Tauri’s macOS signing docs say to use a `Developer ID Application` certificate for shipping outside the App Store, and notarization requires App Store Connect API credentials or Apple ID credentials. The same docs state notarization is required when using a Developer ID Application certificate. ([Tauri][10])

Production environment variables:

```bash
export APPLE_SIGNING_IDENTITY="Developer ID Application: Your Name (TEAMID)"

# App Store Connect API route
export APPLE_API_ISSUER="..."
export APPLE_API_KEY="..."
export APPLE_API_KEY_PATH="/secure/path/AuthKey_XXXX.p8"

# or Apple ID route
export APPLE_ID="you@example.com"
export APPLE_PASSWORD="app-specific-password"
export APPLE_TEAM_ID="TEAMID"

cd apps/desktop
pnpm tauri build --bundles app,dmg
```

For CI, use a macOS runner and `tauri-apps/tauri-action`, but do not block local testing on CI/signing.

## 12. The key product details that will make it feel excellent

1. **Always-visible trust state.** The user should never wonder whether the agent is on host, tmp-copy, or container.

2. **Timeline, not chat.** Chat is only the composer. The main area is an agent activity timeline with tool cards, validation cards, recovery cards, and diff cards.

3. **Bright diffs, collapsible detail.** Every write should produce a visually obvious red/green diff card with file, hunk, receipt, and rollback controls.

4. **Status is first-class.** `/status` should be a live inspector panel, not a text response.

5. **Event stream is structured.** Use `RuntimeEvent`, `WireEnvelope`, and desktop DTOs. Do not scrape terminal text.

6. **Artifacts remain on disk.** The UI is a window into run artifacts, not the only storage system.

7. **No unbounded frontend logs.** Long command output should be chunked, collapsed, and loaded from artifact files on demand.

8. **Use the same slash commands as the CLI.** Desktop command palette should reuse `quorp_slash`/`quorp_term`.

9. **Fast local DMG first.** Use ad-hoc/unsigned builds for internal testing. Sign/notarize only when distributing.

10. **Do not compromise sandbox safety for product polish.** Full-auto on host should remain hard to choose and impossible for yolo-style mode.
