```rust
pub struct ContextRoiSample {
    pub pack_id: PackId,
    pub item_key: String,
    pub cost_tokens: u32,
    pub was_referenced_by_model: bool,
    pub preceded_write_to: Option<PathBuf>,
    pub preceded_validation_delta: ValidationDelta,
    pub repeated_without_new_evidence: bool,
}

pub fn update_roi_score(old: f32, sample: &ContextRoiSample) -> f32 {
    let mut reward = 0.0;
    if sample.was_referenced_by_model {
        reward += 0.25;
    }
    if sample.preceded_write_to.is_some() {
        reward += 0.35;
    }
    if matches!(sample.preceded_validation_delta, ValidationDelta::Improved | ValidationDelta::Green) {
        reward += 0.50;
    }
    if sample.repeated_without_new_evidence {
        reward -= 0.40;
    }

    let token_penalty = (sample.cost_tokens as f32 / 4_000.0).min(0.5);
    (old * 0.85 + (reward - token_penalty) * 0.15).clamp(-1.0, 1.0)
}
```

```rust
pub fn lower_write_file_to_intents(
    path: &Path,
    before: &str,
    after: &str,
) -> anyhow::Result<LoweredEditPlan> {
    let text_diff = diffy::create_patch(before, after);
    let changed_hunks = count_hunks(&text_diff);

    if is_rust(path) {
        if let Some(plan) = lower_rust_ast_edit(path, before, after)? {
            return Ok(plan);
        }
    }

    if is_toml(path) {
        if let Some(plan) = lower_toml_edit(path, before, after)? {
            return Ok(plan);
        }
    }

    if changed_hunks <= 3 {
        return Ok(LoweredEditPlan::from_text_ranges(path, before, after)?);
    }

    anyhow::bail!(
        "broad full-file write rejected: {} changed hunks; use semantic intents or explicit override",
        changed_hunks
    )
}
```

**Runtime injection**

```rust
pub fn select_rules_for_turn(
    cards: &[RuleCard],
    target_path: Option<&Path>,
    failure_kind: Option<&str>,
    budget: u32,
) -> Vec<RuleCard> {
    let mut scored = cards
        .iter()
        .filter(|c| matches!(c.state, RuleLifecycle::Active))
        .filter(|c| c.matches(target_path, failure_kind))
        .map(|c| (c.relevance_score(target_path, failure_kind), c))
        .collect::<Vec<_>>();

    scored.sort_by(|a, b| b.0.total_cmp(&a.0));
    take_until_token_budget(scored, budget)
}
```

**Deny-token-first policy**

```rust
pub fn check_on_surface(&self, action: &Action, surface: ExecutionSurface) -> Decision {
    if self.mode == Mode::YoloSandbox && surface != ExecutionSurface::Sandbox {
        return Decision::Deny;
    }

    if matches!(action.capability, Capability::Read) {
        return Decision::Allow;
    }

    let token_decision = self.check_risky_tokens(action, surface);
    if token_decision != Decision::Allow {
        return token_decision;
    }

    // Only safe, token-free actions reach allowlists.
    if let Some(policy) = self.allow.tools.get(&action.tool_name)
        && policy != &AllowPolicy::Once
    {
        return Decision::Allow;
    }

    if let Some(cmd) = action.command_repr.as_deref()
        && self.compiled_command_globs.iter().any(|(glob, _)| glob.is_match(cmd))
    {
        return Decision::Allow;
    }

    self.default_decision_for_mode(action)
}
```

**Full-auto untrusted repo default**

```text

container or tmp-copy/git-worktree sandbox: required
network: deny by default
dependency install: prompt or deny
MCP/browser: deny by default
generated executable: deny unless built in current run and hash-recorded
project config: may restrict, never elevate
```

**Symlink-safe write guard**

```rust
pub fn canonical_write_target(root: &Path, requested: &Path) -> anyhow::Result<PathBuf> {
    let root = root.canonicalize()?;
    let parent = requested
        .parent()
        .ok_or_else(|| anyhow::anyhow!("write path has no parent"))?;
    let canonical_parent = root.join(parent).canonicalize()?;
    if !canonical_parent.starts_with(&root) {
        anyhow::bail!("write escapes workspace through symlink: {}", requested.display());
    }
    Ok(canonical_parent.join(requested.file_name().unwrap()))
}
```

**No-progress detector**

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProgressSignal {
    pub turn: u64,
    pub new_files_read: usize,
    pub repeated_reads: usize,
    pub writes: usize,
    pub validations: usize,
    pub validation_delta: ValidationDelta,
    pub prompt_tokens: u32,
}

pub fn classify_progress(window: &[ProgressSignal]) -> ProgressState {
    let no_writes = window.iter().all(|s| s.writes == 0);
    let mostly_repeats = window.iter().map(|s| s.repeated_reads).sum::<usize>() >= 3;
    let growing_prompt = window.windows(2).all(|w| w[1].prompt_tokens > w[0].prompt_tokens);

    if no_writes && mostly_repeats && growing_prompt {
        ProgressState::StalledNeedsPatchOrNewEvidence
    } else {
        ProgressState::Healthy
    }
}
```

When stalled, inject a small controller packet:

```text
You have repeated inspection without new evidence.
Choose exactly one:
1. emit a minimal patch,
2. read one specific missing owner range,
3. run one targeted validation.
Do not read broad files again.
```

**Acceptance tests**

2. Repeated reads of the same file range increment no-progress counters.
3. A successful targeted validation resets stall state.
4. A failed validation creates a concise proof packet and negative-memory candidate.
5. `/verify` shows why each validation stage was chosen.

**Expected impact**

This improves coding capability and reduces tokens. The agent stops wandering and gets pushed into evidence-backed action.

```rust
#[derive(Debug, Clone, Default)]
pub struct MissionControlState {
    pub run_id: Option<RunId>,
    pub phase: String,
    pub model_provider: String,
    pub permission_mode: String,
    pub sandbox: SandboxSummary,
    pub context: ContextPressureView,
    pub tasks: Vec<TaskRow>,
    pub agents: Vec<AgentRow>,
    pub proofs: Vec<ProofRow>,
    pub rules: Vec<RuleRow>,
    pub transcript: Vec<TranscriptItem>,
    pub pending_permission: Option<PermissionPrompt>,
}

impl MissionControlState {
    pub fn apply_event(&mut self, event: RuntimeEvent) {
        match event {
            RuntimeEvent::ModelRequestStarted { prompt_token_estimate, .. } => {
                self.context.last_prompt_estimate = prompt_token_estimate;
            }
            RuntimeEvent::ToolCallFinished { action_kind, target_path, status, .. } => {
                self.transcript.push(TranscriptItem::tool(action_kind, target_path, status));
            }
            RuntimeEvent::ValidationFinished { status, summary, .. } => {
                self.proofs.push(ProofRow::validation(summary, status));
            }
            _ => {}
        }
    }
}
```

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct EvalIntegrityReport {
    pub oracle_assisted: bool,
    pub static_case_constants_found: Vec<String>,
    pub exact_patch_embeds_found: Vec<PathBuf>,
    pub replay_deterministic: bool,
    pub artifact_hashes: BTreeMap<PathBuf, String>,
}

pub fn fail_if_public_eval_uses_oracle(report: &EvalIntegrityReport) -> anyhow::Result<()> {
    if report.oracle_assisted
        || !report.static_case_constants_found.is_empty()
        || !report.exact_patch_embeds_found.is_empty()
    {
        anyhow::bail!("public eval is oracle-assisted; score is not publishable");
    }
    Ok(())
}
```

# My final priority order

If I were opening PRs, I would do them in this exact sequence:

1. **Quarantine benchmark oracle playbooks** and ship the general Proof-to-Repair Planner.
2. **Ledger-Native Context OS** with visible token pressure, state capsules, and context ROI.
3. **Semantic Edit Intents** so broad full-file writes become rare, justified, and receipt-backed.
4. **Durable Rule Cards** for fast, scoped, git-trackable learning from mistakes.
5. **Proof-backed Task DAG/Subagents** with worktree isolation and merge-by-receipt.
6. **Permission vNext** with deny-token-first semantics, receipts, and symlink-safe boundaries.
7. **Proof-Driven Validation Planner** with no-progress detection.
8. **Mission-Control TUI** that exposes Quorp’s hidden strengths.
9. **Replay/eval scoreboard** that proves improvements honestly.

The most novel combination is **Context OS + Rule Cards + Proof-backed Task DAG + Mission-Control TUI**. That would make Quorp feel less like a chat CLI and more like an auditable autonomous engineering operating system.
