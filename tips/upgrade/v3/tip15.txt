I could inspect the public GitHub tree and archived run artifacts, but not run a local clone/test pass from this sandbox because GitHub DNS resolution failed here. The recommendations below are therefore evidence-grounded from the repo/logs and current public sources, not from executing Quorp locally.

The most important finding: **Quorp’s 5/5 Rust SWE-bench-top5 run is valuable behavioral evidence, but it is not yet proof of general autonomous coding strength.** The archive reports 5/5 successes, 28 requests, 180,546 billed tokens, and about 696 seconds wall-clock, but the runtime also contains benchmark-specific deterministic repair playbooks and the logs show controller-injected deterministic patches for at least the cargo-dist and cc-rs cases. use serde::{Deserialize, Serialize};
```rust
use std::path::PathBuf;

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
pub enum RepairDirective {
    /// The agent has insufficient evidence. Force one targeted observation,
    /// not another broad repo crawl.
    TargetedRead {
        path: PathBuf,
        range: Option<LineRange>,
        reason: String,
        evidence_refs: Vec<EvidenceRef>,
    },

    /// The model produced invalid JSON/prose. Retry with a smaller action contract,
    /// preserving objective, failing command, suspected files, and invalid-output digest.
    ContractRetry {
        failure_class: FailureClass,
        compact_contract: ActionContract,
        forbidden_repeats: Vec<ActionFingerprint>,
    },

    /// The state has enough evidence for an edit. Require a patch plan with hashes,
    /// validation commands, and rollback point.
    RequirePatchPlan {
        target_files: Vec<FileLease>,
        validation: ValidationPlan,
        max_files: usize,
        reason: String,
    },

    /// No progress, stale target, or unsafe behavior.
    Backtrack {
        checkpoint: CheckpointId,
        reason: String,
        preserve: Vec<EvidenceRef>,
    },
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
pub struct EvidenceRef {
    pub ledger_seq: u64,
    pub kind: EvidenceKind,
    pub digest: blake3::Hash,
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
pub struct LineRange {
    pub start: u32,
    pub end: u32,
}
```

```rust
// crates/quorp_repair/src/kernel.rs

pub struct RepairKernel<I, M, R> {
    pub index: I,
    pub memory: M,
    pub rules: R,
    pub limits: RepairLimits,
}

impl<I, M, R> RepairKernel<I, M, R>
where
    I: ContextIndex,
    M: NegativeMemoryStore,
    R: RuleSelector,
{
    pub fn next_directive(
        &self,
        snapshot: &AgentTaskStateSnapshot,
        failure: &FailurePacket,
        evidence: &EvidenceBundle,
    ) -> anyhow::Result<Option<RepairDirective>> {
        if let Some(repeat) = self.memory.detect_repeat_failure(failure)? {
            return Ok(Some(RepairDirective::Backtrack {
                checkpoint: repeat.last_good_checkpoint,
                reason: format!("repeated failure: {}", repeat.failure_class),
                preserve: repeat.evidence_refs,
            }));
        }

        if failure.is_parser_failure() {
            return Ok(Some(RepairDirective::ContractRetry {
                failure_class: failure.class.clone(),
                compact_contract: ActionContract::minimal_repair(snapshot, evidence),
                forbidden_repeats: snapshot.recent_action_fingerprints(),
            }));
        }

        let candidate_files = self.index.rank_repair_targets(failure, evidence)?;
        if candidate_files.is_empty() {
            return Ok(None);
        }

        if evidence.is_patch_ready(&candidate_files) {
            return Ok(Some(RepairDirective::RequirePatchPlan {
                target_files: candidate_files.into_iter().take(3).collect(),
                validation: ValidationPlan::from_failure(failure),
                max_files: 3,
                reason: "ranked failure evidence is sufficient for a minimal patch".to_string(),
            }));
        }

        Ok(Some(RepairDirective::TargetedRead {
            path: candidate_files[0].path.clone(),
            range: candidate_files[0].suspected_range,
            reason: candidate_files[0].why.clone(),
            evidence_refs: candidate_files[0].evidence_refs.clone(),
        }))
    }
}
```

### 2. Build a ledger-backed Context OS with lossless compaction and tool-result handles

**Why this is #2:** It directly attacks token waste, context drift, resumability, and long-form reliability.

```rust
// crates/quorp_agent_core/src/runtime/write_readiness.rs

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum WriteReadiness {
    NotReady,
    NeedsOneTargetedRead,
    ReadyForPreviewPatch,
    ReadyForHashCheckedReplace,
    BlockedBySafety,
}

#[derive(Debug, Clone)]
pub struct WriteReadinessReport {
    pub readiness: WriteReadiness,
    pub suspected_files: Vec<SuspectedFile>,
    pub missing_evidence: Vec<MissingEvidence>,
    pub forbidden_next_actions: Vec<ActionKind>,
    pub reason: String,
}

pub fn assess_write_readiness(
    state: &AgentTaskStateSnapshot,
    failure: Option<&FailurePacket>,
    index_hits: &[ContextHit],
    recent_actions: &[ActionRecord],
) -> WriteReadinessReport {
    if state.security_boundary_blocks_write() {
        return WriteReadinessReport::blocked("current permission/sandbox state blocks writes");
    }

    let repeated_reads = recent_actions
        .iter()
        .rev()
        .take(5)
        .filter(|a| matches!(a.kind, ActionKind::ReadFile | ActionKind::Search))
        .count();

    let has_failing_signal = failure.is_some() || state.last_failing_verifier.is_some();
    let has_target = !index_hits.is_empty() || !state.working_set.is_empty();
    let has_hash = state.working_set.iter().any(|f| f.last_read_hash.is_some());

    if has_failing_signal && has_target && has_hash {
        return WriteReadinessReport {
            readiness: WriteReadiness::ReadyForPreviewPatch,
            suspected_files: SuspectedFile::from_hits(index_hits),
            missing_evidence: vec![],
            forbidden_next_actions: if repeated_reads >= 3 {
                vec![ActionKind::Search, ActionKind::ReadFile]
            } else {
                vec![]
            },
            reason: "failure signal, target file, and hash evidence are available".into(),
        };
    }

    if has_failing_signal && has_target {
        return WriteReadinessReport::needs_one_read("target exists but hash/range evidence is missing");
    }

    WriteReadinessReport::not_ready("missing failure signal or plausible target file")
}
```

```rust
// crates/quorp_agent_core/src/runtime/rule_gate.rs

pub struct RuleGate<S> {
    store: S,
    max_rules: usize,
    max_tokens: usize,
}

impl<S: RuleStore> RuleGate<S> {
    pub fn select_for_turn(
        &self,
        state: &AgentTaskStateSnapshot,
        action_contract: &ActionContract,
        failure: Option<&FailurePacket>,
    ) -> anyhow::Result<SelectedRules> {
        let mut candidates = self.store.matching_rules(
            RuleMatchInput {
                paths: state.working_paths(),
                tools: action_contract.allowed_tools(),
                failure_class: failure.map(|f| f.class.clone()),
                permission_mode: state.permission_mode(),
                repo_trust: state.repo_trust(),
            }
        )?;

        candidates.retain(|r| !r.safety.may_grant_permissions);
        candidates.sort_by_key(|r| std::cmp::Reverse(r.expected_value_score()));

        Ok(SelectedRules::bounded(
            candidates,
            self.max_rules,
            self.max_tokens,
        ))
    }
}
```

```rust
// crates/quorp_patch_vm/src/patch_tournament.rs

pub fn choose_patch(candidates: Vec<PatchCandidate>) -> anyhow::Result<PatchWinner> {
    let mut scored = Vec::new();

    for candidate in candidates {
        let score = PatchScore {
            deterministic_pass: candidate.proof.deterministic_pass,
            hidden_risk: candidate.risk.estimated_hidden_test_risk,
            changed_lines: candidate.diff.changed_lines(),
            changed_files: candidate.diff.changed_files(),
            permission_cost: candidate.permission_cost,
            rollback_clean: candidate.rollback_receipt.is_some(),
        };

        if score.deterministic_pass {
            scored.push((candidate, score));
        }
    }

    scored.sort_by_key(|(_, s)| {
        (
            std::cmp::Reverse(s.deterministic_pass),
            s.permission_cost,
            s.changed_files,
            s.changed_lines,
            s.hidden_risk.milli_score(),
        )
    });

    scored
        .into_iter()
        .next()
        .map(|(candidate, score)| PatchWinner { candidate, score })
        .ok_or_else(|| anyhow::anyhow!("no candidate patch passed deterministic validation"))
}
```

```rust
// crates/quorp_permissions/src/capability.rs

#[derive(Debug, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub enum CapabilityToken {
    ShellMeta,
    CompoundCommand,
    NetworkEgress,
    DependencyInstall,
    GeneratedExecutable,
    SecretRead,
    SecretWrite,
    FilesystemDelete,
    FilesystemOutsideWorkspace,
    GitRemoteWrite,
    ProcessKill,
    BrowserEgress,
    McpTool { server: String, tool: String },
    SandboxEscapeRisk,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum PermissionDecision {
    Allow { reason: String },
    Ask { reason: String, tokens: Vec<CapabilityToken> },
    Deny { reason: String, tokens: Vec<CapabilityToken> },
    DeferToHook { hook_id: String },
}

pub fn classify_shell_command(cmd: &str, trust: &TrustContext) -> PermissionDecision {
    let ast = match ShellAst::parse(cmd) {
        Ok(ast) => ast,
        Err(err) => {
            return PermissionDecision::Ask {
                reason: format!("could not parse shell command safely: {err}"),
                tokens: vec![CapabilityToken::ShellMeta],
            };
        }
    };

    let tokens = ast.capability_tokens();

    if tokens.contains(&CapabilityToken::SandboxEscapeRisk) {
        return PermissionDecision::Deny {
            reason: "command has sandbox escape risk".into(),
            tokens,
        };
    }

    if tokens.contains(&CapabilityToken::NetworkEgress) && !trust.network_allowed {
        return PermissionDecision::Ask {
            reason: "network egress is disabled for this repo/session".into(),
            tokens,
        };
    }

    if tokens.contains(&CapabilityToken::DependencyInstall) && !trust.dependency_install_allowed {
        return PermissionDecision::Ask {
            reason: "dependency install requires approval".into(),
            tokens,
        };
    }

    if tokens.iter().any(|t| matches!(
        t,
        CapabilityToken::SecretRead
            | CapabilityToken::GitRemoteWrite
            | CapabilityToken::FilesystemOutsideWorkspace
    )) {
        return PermissionDecision::Ask {
            reason: "command touches sensitive capability".into(),
            tokens,
        };
    }

    PermissionDecision::Allow {
        reason: "command fits current sandbox/trust policy".into(),
    }
}
```

### 8. Make Patch VM the only normal write path, with semantic diff receipts and broad-write exceptions

**Why this is #8:** It improves safety, auditability, and token efficiency.

The support docs say Quorp has Patch VM-backed write receipts and proof packets. ([GitHub][3]) The successful run also involved broad full-file controller writes in benchmark repair cases; cargo-dist wrote six files and cc-rs wrote `src/lib.rs`, with the cc-rs benchmark report showing `lines_added=65`, `lines_removed=40`, and write count 1, while the event log shows a large full-file write. **Upgrade:** Normal runtime writes must go through Preview/Edit/ReplaceRange/Patch VM with hash preconditions. Full-file `WriteFile` should require a specific exception: generated file, new file, formatter rewrite, or user-approved broad write.

## The highest-leverage combined architecture

The strongest version of Quorp is not “Claude Code but Rust.” It is:

```text
Ledger-native agent runtime
  → Context OS with lossless compaction
  → Repair Kernel with negative memory
  → Task DAG with proof-backed subagents
  → Patch VM as exclusive write contract
  → Deterministic permission autopilot
  → Mission Control TUI
  → Replay/eval matrix
```

That combination creates capabilities that are hard for competitors to match:

1. **Every decision is replayable.**
2. **Every patch has a proof receipt.**
3. **Every long session has visible context pressure and preserved state.**
4. **Every mistake can become a scoped rule or negative memory.**
5. **Every subagent return is patch/proof-backed, not vibes.**
6. **Every permission decision is inspectable before side effects.**
7. **Every eval can be replayed, perturbed, and audited for oracle leakage.**

## My strongest recommendation for the first PR sequence

Do these in order:

1. **Anti-oracle quarantine PR**
   Move `benchmark_playbooks.rs` out of normal runtime, add `script/quorp-anti-oracle-audit`, and fail CI if benchmark patches are reachable from `quorp_agent_core`.

2. **Context budget/status PR**
   Add `ContextBudgetFrame`, ledger events for token pressure, and `/status` context pressure. No full compaction yet; just visibility.

3. **Tool-result handle PR**
   Store large tool outputs in ledger/artifacts and prompt compact handles instead of raw long text.

4. **First-write accelerator PR**
   Add `WriteReadinessReport` and block repeated broad reads once patch evidence is sufficient.

5. **Rule Forge MVP PR**
   Add `agent/rules/{candidate,shadow,active,rejected,deprecated}`, schema validation, negative memory, and `/rules`.

6. **Mission Control MVP PR**
   Build `/status`, `/context`, `/diff`, `/proof`, `/permissions`, and `/tasks` from one typed `MissionControlState`.

That sequence gives immediate credibility, cuts tokens, improves autonomy, and starts building the uniquely powerful Quorp experience.
