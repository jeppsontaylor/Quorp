## Scope note

I could not run a fresh local clone/test execution from this environment, so I am **not** claiming I executed Quorp locally. I inspected the public repo, raw source/log artifacts, and current public docs/research. The conclusions below are strongest where backed by the 2026-04-28 run archive and repo source.

My hard read: **Quorp’s winning path is not “more prompts” or a prettier clone of Claude Code/Codex. It should become a ledger-native, proof-backed autonomous coding OS: every task, context pack, rule, permission, patch, validation, rollback, and replay is typed, inspectable, resumable, and evaluable.**

The 5/5 Rust run is useful evidence, but not sufficient proof of generality: the archive reports 5/5 successes, 28 requests, 180,546 billed tokens, and about 696 seconds wall time; it also shows delayed first writes, parser recovery in three cases, broad full-file writes, and benchmark playbook/oracle behavior in the harder cases. ([GitHub][1]) Current public competitors already have serious primitives: Codex has sandboxing/approval controls and subagents; Claude Code has permission modes, hooks, slash commands, and auto memory; mini-SWE-agent shows a very simple loop can still score strongly on SWE-bench Verified, so Quorp must win through **state, proof, safety, and replay**, not scaffold complexity alone. ([OpenAI Developers][2])

# Final ranked top 10 ideas

## 1. Quarantine benchmark-oracle repairs and replace them with a general Repair Synthesis Engine

**Why this is #1:** Quorp’s biggest credibility and capability risk is that parts of the successful run are not general agent behavior. The current `runtime/benchmark_playbooks.rs` is explicitly benchmark-specific and includes exact deterministic source patches gated by `PolicyMode::BenchmarkAutonomous`; the cargo-dist and cc-rs cases show controller-injected deterministic source patches after parser recovery. ([GitHub][3])

**Best upgrade:** move all exact benchmark playbooks out of the runtime and into a quarantined eval-oracle crate. Then build a general repair engine that can synthesize candidate patches from observed failures, owner slices, diagnostics, proof packets, and Patch VM constraints, without reading benchmark golden patches.

**Files to change/add:**

* Move/quarantine:

  * `crates/quorp_agent_core/src/runtime/benchmark_playbooks.rs`
  * to something like `crates/quorp_benchmark/src/oracle_playbooks.rs`
* Add:

  * `crates/quorp_agent_core/src/runtime/repair_synthesis.rs`
  * `crates/quorp_agent_core/src/runtime/repair_source.rs`
  * `crates/quorp_benchmark/src/oracle_policy.rs`
* Modify:

  * `crates/quorp_agent_core/src/runtime.rs`
  * `crates/quorp_agent_core/src/runtime/recovery.rs`
  * `crates/quorp_agent_core/src/ledger.rs`
  * `script/quorp-benchmark-regression-gate`

**Hard implementation shape:**

```rust

    Model,
    ControllerGeneral,
    EvalOracle,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RepairCandidate {
    pub source: RepairSource,
    pub actions: Vec<AgentAction>,
    pub rationale: String,
    pub evidence: Vec<ProofRef>,
    pub expected_preconditions: Vec<PatchPrecondition>,
}

pub fn accept_repair_candidate(
    state: &mut AgentTaskState,
    candidate: RepairCandidate,
) -> Result<Option<Vec<AgentAction>>> {
    match candidate.source {
        RepairSource::EvalOracle if !state.policy.allow_eval_oracle => {
            state.emit(RuntimeEvent::RepairSuppressed {
                source: RepairSource::EvalOracle,
                reason: "eval oracle repair is forbidden outside oracle benchmark lane".into(),
            });
            return Ok(None);
        }
        RepairSource::EvalOracle => {
            state.scoreboard_flags.non_comparable = true;
            state.emit(RuntimeEvent::BenchmarkOracleUsed {
                reason: "oracle repair used; result excluded from public capability score".into(),
            });
        }
        RepairSource::Model | RepairSource::ControllerGeneral => {
            require_hash_preconditions(&candidate.actions)?;
            require_patch_vm_route(&candidate.actions)?;
        }
    }

    Ok(Some(candidate.actions))
}
```

**Acceptance tests:**

* `normal_run_never_uses_eval_oracle`
* `benchmark_oracle_success_marked_non_comparable`
* `runtime_cannot_read_upstream_test_patch`
* `repair_synthesis_requires_hash_preconditions`
* rerun top-5 in three lanes:

  * model-only
  * general-controller
  * oracle-comparison, clearly non-comparable

**Expected impact:** This may initially lower benchmark score, but it will make every future score meaningful. It also forces Quorp to learn general repair behaviors instead of silently carrying benchmark-specific intelligence.

## 3. Replace broad `write_file` behavior with a Patch Delta VM

**Why this is #3:** Quorp’s current write behavior is too permissive for high-agency autonomy. The run includes broad writes: chrono wrote `src/round.rs` as an 805-line file for a tiny removal; axum wrote a 705-line file; cc-rs wrote a 4,278-line file. ([GitHub][6]) The roadmap already calls out repo-wide Patch VM write-path enforcement as open work. ([GitHub][4])

**Best upgrade:** `write_file` should become a last-resort primitive. Normal runs should route all nontrivial writes through minimal range hunks with old-hash preconditions, proof receipts, rollback handles, and broad-write risk scoring.

**Files to change/add:**

* Add:

  * `crates/quorp_patch_model/src/delta_model.rs`
* Modify:

  * `crates/quorp_patch_vm`
  * `crates/quorp_tools`
  * `crates/quorp_agent_core/src/runtime/write_routing.rs`
  * native backend action mirrors mentioned by the repo’s audit tooling
  * `script/quorp-write-path-audit`

**Hard implementation shape:**

```rust
pub struct WritePolicy {
    pub max_full_write_bytes: usize,
    pub max_changed_lines_without_review: usize,
    pub require_hash_precondition: bool,
    pub allow_broad_write: bool,
}

pub fn rewrite_write_file_to_delta(
    path: &Utf8Path,
    old_text: &str,
    new_text: &str,
    policy: &WritePolicy,
) -> Result<PatchPlan> {
    let old_hash = ContentHash::sha256(old_text);
    let hunks = myers_range_hunks(old_text, new_text)?;

    let changed = hunks.iter().map(|h| h.changed_line_count()).sum::<usize>();
    let broad = new_text.len() > policy.max_full_write_bytes
        || changed > policy.max_changed_lines_without_review;

    if broad && !policy.allow_broad_write {
        bail!(
            "broad write rejected: {path}; changed_lines={changed}; \
             use range edits or request explicit review"
        );
    }

    Ok(PatchPlan::RangeEdits {
        path: path.to_owned(),
        expected_old_hash: old_hash,
        hunks,
        rollback: RollbackPlan::from_old_text(old_hash, old_text),
    })
}
```

**Acceptance tests:**

* `large_write_converted_to_range_hunks`
* `large_write_rejected_without_review`
* `stale_hash_aborts_patch`
* `rollback_restores_original_hash`
* `write_receipt_contains_old_new_hashes`
* `benchmark_oracle_cannot_bypass_patch_vm`

**Expected impact:** fewer regressions, easier code review, better auditability, and much higher trust in full-auto mode.

pub fn select_rules_for_turn(
    store: &RuleStore,
    ctx: &TurnContext,
    budget: RuleBudget,
    trust: &TrustState,
) -> Result<Vec<RuleCard>> {
    let mut cards = store.query(ctx.scope())?;

    cards.retain(|card| {
        card.state.is_prompt_eligible()
            && card.safety.allowed_under(trust)
            && !card.safety.can_elevate_permissions
            && card.matches(ctx)
    });

    cards.sort_by_key(|card| Reverse(card.utility_score(ctx)));
    cards.truncate(budget.max_rules_per_turn);

    Ok(cards)
}

bitflags::bitflags! {
    pub struct Capability: u64 {
        const READ_WORKSPACE        = 1 << 0;
        const WRITE_WORKSPACE       = 1 << 1;
        const COMPOUND_SHELL        = 1 << 2;
        const NETWORK               = 1 << 3;
        const DEPENDENCY_INSTALL    = 1 << 4;
        const GENERATED_EXECUTABLE  = 1 << 5;
        const SECRET_READ           = 1 << 6;
        const GIT_REMOTE_WRITE      = 1 << 7;
        const MCP                   = 1 << 8;
        const BROWSER               = 1 << 9;
        const FIND_DELETE           = 1 << 10;
        const FIND_EXEC             = 1 << 11;
    }
}

pub fn classify_shell_command(cmd: &str) -> Capability {
    let mut caps = Capability::empty();

    if contains_shell_meta(cmd, &["&&", "||", ";", "|", ">", "<", "$(", "`"]) {
        caps |= Capability::COMPOUND_SHELL;
    }
    if contains_any(cmd, &["curl ", "wget ", "gh api", "git push", "ssh "]) {
        caps |= Capability::NETWORK;
    }
    if contains_any(cmd, &["cargo install", "npm install", "pnpm add", "pip install"]) {
        caps |= Capability::DEPENDENCY_INSTALL | Capability::NETWORK;
    }
    if contains_any(cmd, &["find "]) && contains_any(cmd, &[" -delete", " -exec"]) {
        caps |= Capability::FIND_DELETE | Capability::FIND_EXEC;
    }
    if likely_executes_recent_build_artifact(cmd) {
        caps |= Capability::GENERATED_EXECUTABLE;
    }

    caps
}
```

**Acceptance tests:**

**Expected impact:** safer full-auto autonomy, fewer scary prompts, clearer audit logs, and better trust than blind “YOLO” modes.

pub enum CompileOutcome {
    Valid(Vec<AgentAction>),
    Repaired {
        actions: Vec<AgentAction>,
        repair: ProtocolRepairReceipt,
    },
    NeedModelRetry {
        packet: RecoveryPacket,
    },
    Reject {
        reason: ProtocolRejectReason,
    },
}

pub fn compile_agent_turn(raw: &str, schema: &ActionSchema) -> CompileOutcome {
    if let Ok(actions) = parse_strict_actions(raw, schema) {
        return CompileOutcome::Valid(actions);
    }

    match bounded_json_repair(raw) {
        Some(repaired) if repair_is_syntax_only(raw, &repaired) => {
            match parse_strict_actions(&repaired, schema) {
                Ok(actions) if actions_are_low_risk(&actions) => {
                    CompileOutcome::Repaired {
                        actions,
                        repair: ProtocolRepairReceipt::syntax_only(raw, repaired),
                    }
                }
                _ => CompileOutcome::NeedModelRetry {
                    packet: RecoveryPacket::minimal_schema_error(raw, schema),
                },
            }
        }
        _ => CompileOutcome::NeedModelRetry {
            packet: RecoveryPacket::minimal_schema_error(raw, schema),
        },
    }
}
```

## 10. Create an anti-overfit evaluation program with replay scoreboards

**Why this matters:** SWE-bench Verified is a human-validated 500-instance subset and remains important, but benchmark comparisons are fragile because scaffolds differ; SWE-rebench explicitly targets continuous/decontaminated tasks, and test-overfitting research warns that passing observed tests can still break hidden behavior. ([SWE-bench][15]) Terminal-Bench 2.0 adds 89 terminal-agent tasks across software engineering, ML, security, data science, and more, which is a better fit for Quorp’s terminal-first ambition. ([Terminal-Bench][16])

**Best upgrade:** every Quorp release should publish separate score lanes:

1. **Oracle-free capability score**
2. **General-controller score**
3. **Eval-oracle diagnostic score**, explicitly non-comparable
4. **Secure full-auto score**
5. **Token efficiency score**
6. **Patch quality score**
7. **Replay determinism score**
8. **Context compaction score**
9. **Rule learning helped/hurt score**

**Files to change/add:**

* Add:

  * `crates/quorp_benchmark/src/score_lanes.rs`
  * `crates/quorp_benchmark/src/replay_scoreboard.rs`
  * `crates/quorp_benchmark/src/anti_overfit.rs`
  * `benchmark/suites/security/`
  * `benchmark/suites/context-stress/`
  * `benchmark/suites/rule-learning/`
* Modify:

  * `crates/quorp_cli/src/quorp/benchmark`
  * `script/quorp-benchmark-regression-gate`
  * `justfile`

**Hard implementation shape:**

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CapabilityScore {
    pub solved: bool,
    pub oracle_free: bool,
    pub secure_full_auto: bool,
    pub deterministic_replay: bool,
    pub token_billed: u64,
    pub requests: u32,
    pub first_write_turn: Option<u32>,
    pub broad_write_risk: BroadWriteRisk,
    pub parser_recoveries: u32,
    pub rule_helped: i32,
    pub context_compactions: u32,
    pub proof_completeness: f32,
}
```

**Acceptance tests/gates:**

* any `RepairSource::EvalOracle` marks run non-comparable
* broad write over threshold fails patch-quality lane
* replay must reproduce task DAG, context packs, proof receipts
* untrusted full-auto suite must not touch host/network/secrets
* context stress suite must remain under token budget
* rule-learning suite must prove rules help in replay before promotion

**Expected impact:** Quorp can claim improvement credibly, avoid overfitting, and catch regressions in the exact mechanisms that made the 5/5 run succeed.

# The highest-leverage implementation order

**First 1–3 days:** quarantine benchmark playbooks, add `RepairSource`, mark oracle usage non-comparable, add context budget events, and block/convert broad `write_file` writes above a threshold.

**First 1–2 weeks:** implement `StatePackV2`, `/status` context pressure, Patch Delta VM receipts, command capability classification, and parser Action Compiler.

**First 30 days:** ProofLens, RuleCards lifecycle, task DAG, subagent worktrees, and mission-control TUI panes.

**Moonshot:** make Quorp’s ledger the product: a replayable debugger for agent cognition where every context decision, rule trigger, patch, proof, permission, failure, rollback, and subagent return is inspectable and reproducible. That is the capability Claude CLI and Codex CLI do not yet put at the center.
