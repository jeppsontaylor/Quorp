```rust
pub fn evaluate_contract(
    contract: &ActionContract,
    actions: &[AgentAction],
    state: &AgentTaskState,
) -> ContractDecision {
    if let Some(required) = &contract.required_next {
        if !actions.iter().any(|a| required.matches(a)) {
            return ContractDecision::RejectWithRecovery {
                reason: format!("required next action was not honored: {required:?}"),
                recovery_packet: RecoveryPacket::from_contract(contract, state),
            };
        }
    }

    for action in actions {
        for forbidden in &contract.forbidden {
            if forbidden.matches(action) {
                return ContractDecision::RejectWithRecovery {
                    reason: format!("action violates active contract: {forbidden:?}"),
                    recovery_packet: RecoveryPacket::from_contract(contract, state),
                };
            }
        }
    }

    ContractDecision::Allow
}
```

```rust
// crates/quorp_prompt/src/block.rs

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct PromptBlock {
    pub id: String,
    pub kind: PromptBlockKind,
    pub trust: PromptTrust,
    pub priority: PromptPriority,
    pub cache_policy: CachePolicy,

    /// Stable hash of rendered body.
    pub content_hash: String,

    /// Estimated local tokens and provider-reported tokens if available.
    pub estimated_tokens: u32,
    pub provider_tokens: Option<u32>,

    /// Whether compaction may summarize this block.
    pub compaction: CompactionPolicy,

    pub body: String,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum PromptBlockKind {
    SystemContract,
    ToolSchema,
    StateKernel,
    TaskDag,
    ActiveRules,
    ContextPack,
    ValidationDelta,
    RecoveryPacket,
    TranscriptTail,
    UserInstruction,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct PromptPlan {
    pub plan_id: String,
    pub model_context_limit: u32,
    pub reserved_output_tokens: u32,
    pub blocks: Vec<PromptBlock>,
}

impl PromptPlan {
    pub fn render_full(&self) -> RenderedPrompt {
        RenderedPrompt {
            text: self.blocks.iter().map(|b| b.body.as_str()).collect::<Vec<_>>().join("\n\n"),
            block_stats: self.stats(),
        }
    }

    pub fn render_delta(&self, previous: Option<&PromptPlan>) -> RenderedPrompt {
        let Some(previous) = previous else {
            return self.render_full();
        };

        let previous_hashes = previous
            .blocks
            .iter()
            .map(|b| (b.id.as_str(), b.content_hash.as_str()))
            .collect::<std::collections::BTreeMap<_, _>>();

        let mut body = String::new();
        for block in &self.blocks {
            let unchanged = previous_hashes
                .get(block.id.as_str())
                .is_some_and(|hash| *hash == block.content_hash);

            match (unchanged, &block.cache_policy) {
                (true, CachePolicy::ProviderCached) => {
                    body.push_str(&format!("\n\n[unchanged-block:{}:{}]", block.id, block.content_hash));
                }
                _ => {
                    body.push_str("\n\n");
                    body.push_str(&block.body);
                }
            }
        }

        RenderedPrompt {
            text: body,
            block_stats: self.stats(),
        }
    }
}
```

```rust
pub fn load_rule_file(path: &Path, repo_trust: RepoTrust) -> anyhow::Result<LoadedRule> {
    let rule: RuleFile = serde_json::from_str(&std::fs::read_to_string(path)?)?;

    if rule.safety.may_relax_permissions {
        anyhow::bail!("rule {} rejected: rules may not relax permissions", rule.id);
    }

    if matches!(repo_trust, RepoTrust::Untrusted)
        && matches!(rule.status, RuleStatus::Approved | RuleStatus::Active)
    {
        return Ok(LoadedRule::shadow_only(
            rule,
            "repo-authored approved rules are shadow-only until locally trusted",
        ));
    }

    if rule.effect.prompt_token_estimate() > rule.safety.max_prompt_tokens {
        anyhow::bail!("rule {} rejected: exceeds max_prompt_tokens", rule.id);
    }

    Ok(LoadedRule::active(rule))
}
```

## 7. Capability Firewall for full-auto mode

Quorp already has a permissions engine with modes, command token classification, network/dependency-install/secret/generated-executable detection, and `yolo` restricted to sandbox surface.  It also has sandboxing via tmp-copy/git-worktree and container runtime support with `--network=none`, dropped capabilities, and `no-new-privileges`.  That is a strong base. The next step is a **capability firewall** that is visible, auditable, and resistant to repo-authored escalation.

OpenAI Codex CLI’s documented modes include default suggest, auto-edit, and full-auto in a sandboxed, network-disabled environment. ([OpenAI Help Center][6]) Claude Code has fine-grained permissions, deny/ask/allow rules, and auto-mode-style policy surfaces. ([Claude][7]) Quorp should go stricter for untrusted repos.

**Files to add/modify**

```text

crates/quorp_permissions/src/trust_policy.rs
crates/quorp_sandbox/src/network.rs
crates/quorp_sandbox/src/secrets.rs
crates/quorp_tools/src/command_policy.rs
crates/quorp_render/src/components/permission_timeline.rs
```

**Policy model**

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CapabilityPolicy {
    pub repo_trust: RepoTrust,
    pub execution_surface: ExecutionSurface,
    pub network: CapabilityMode,
    pub dependency_install: CapabilityMode,
    pub mcp: CapabilityMode,
    pub browser: CapabilityMode,
    pub generated_executable: CapabilityMode,
    pub secret_read: CapabilityMode,
    pub destructive_fs: CapabilityMode,
}

#[derive(Debug, Clone, Copy, Serialize, Deserialize)]
pub enum CapabilityMode {
    Allow,
    Ask,
    Deny,
}

pub fn decide(policy: &CapabilityPolicy, action: &Action) -> CapabilityDecision {
    if action.tokens.contains(&CapabilityToken::SecretsRead) {
        return CapabilityDecision::Deny("secret-like path read blocked".into());
    }

    if action.tokens.contains(&CapabilityToken::Network)
        && !matches!(policy.network, CapabilityMode::Allow)
    {
        return CapabilityDecision::AskOrDeny(policy.network, "network access".into());
    }

    if action.tokens.contains(&CapabilityToken::DependencyInstall)
        && matches!(policy.repo_trust, RepoTrust::Untrusted)
    {
        return CapabilityDecision::Ask("dependency install in untrusted repo".into());
    }

    CapabilityDecision::Allow
}
```

## 9. Semantic Rust editing tools through Patch VM

Quorp’s Patch VM already validates preconditions, previews changes, applies with rollback, and emits receipts.  The context index already records Rust symbols and tests.  The next capability jump is to let the model edit by symbol intent rather than raw file text.

**Files to add**

```text
crates/quorp_rust_intel/
  Cargo.toml
  src/lib.rs
  src/symbol_edit.rs
  src/rust_analyzer.rs
  src/tree_sitter.rs

crates/quorp_patch_vm/src/symbol_patch.rs
crates/quorp_agent_core/src/tools/semantic_edit.rs
```

Add `AgentAction` variants:

```rust
AgentAction::EditSymbol {
    symbol: String,
    operation: SymbolEditOperation,
    expected_definition_hash: String,
}

AgentAction::InsertIntoImpl {
    impl_symbol: String,
    method_signature: String,
    body: String,
    expected_impl_hash: String,
}
```

**Why it matters**

Instead of reading a whole file and writing it back, the model can request:

```json
{
  "action": "edit_symbol",
  "symbol": "Router::merge",
  "operation": {
    "replace_match_arm": {
      "pattern": "(Fallback::Custom(_), Fallback::Custom(_))",
      "body": "panic!(...)"
    }
  },
  "expected_definition_hash": "..."
}
```

The backend resolves the symbol, constructs a Patch VM change with hash preconditions, and rejects if the symbol moved.

**Acceptance tests**

```text
cargo test -p quorp_rust_intel resolves_rust_symbol_definition

cargo test -p quorp_agent_core edit_symbol_emits_patch_receipt
cargo test -p quorp_agent_core semantic_edit_does_not_need_full_file_prompt
```

This directly improves coding capability and token economy.
